## Prerequisites

Before you begin, make sure you have:

- An **Alfred Black account** with an active subscription at [app.alfred.black](https://app.alfred.black)
- An **API key** (we'll create one in Step 1)
- `curl` or any HTTP client

## Step 1: Create an API key

Go to your [Alfred Black dashboard](https://app.alfred.black), navigate to **Settings → API Keys**, click **Create**, give it a name, and copy the key. It starts with `alf_` and is shown only once.

<Warning>
  Copy your API key immediately — you won't be able to see it again. Store it somewhere safe.
</Warning>

## Step 2: Test connectivity

Confirm your API key works by checking your instance health:

<CodeGroup>
```bash cURL
curl -s https://app.alfred.black/api/v1/admin/health \
  -H "Authorization: Bearer alf_your_key_here" | jq .
```

```javascript JavaScript
const res = await fetch('https://app.alfred.black/api/v1/admin/health', {
  headers: { 'Authorization': 'Bearer alf_your_key_here' }
});
console.log(await res.json());
```

```python Python
import requests

res = requests.get(
    'https://app.alfred.black/api/v1/admin/health',
    headers={'Authorization': 'Bearer alf_your_key_here'}
)
print(res.json())
```
</CodeGroup>

You should see a JSON response with health status information.

## Step 3: Explore your vault schema

See what record types Alfred manages:

```bash
curl -s https://app.alfred.black/api/v1/vault/schema \
  -H "Authorization: Bearer alf_your_key_here" | jq .known_types
```

This returns all 19 record types -- from `person` and `project` to `assumption` and `decision`.

## Step 4: Drop your first item into the inbox

Upload a meeting note to the inbox. You can do this from your dashboard (**Inbox** page) or via the API:

```bash
curl -X POST https://app.alfred.black/api/v1/vault/inbox \
  -H "Authorization: Bearer alf_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{
    "filename": "standup-2026-02-26.md",
    "content": "# Daily Standup - Feb 26\n\nAttendees: Alice, Bob, Carol\n\n## Alice\n- Finished the auth migration (project: Platform v2)\n- Blocked on API rate limiting design\n- Decision: use token bucket algorithm\n\n## Bob\n- Reviewed PR #847 for Carol\n- Starting database indexing task\n\n## Carol\n- PR #847 ready for merge\n- Meeting with Design team tomorrow about dashboard redesign"
  }'
```

## Step 5: Check what the Curator produced

Wait a few seconds for the Curator to process, then list records:

<CodeGroup>
```bash List people
curl -s https://app.alfred.black/api/v1/vault/list/person \
  -H "Authorization: Bearer alf_your_key_here" | jq .
```

```bash List tasks
curl -s https://app.alfred.black/api/v1/vault/list/task \
  -H "Authorization: Bearer alf_your_key_here" | jq .
```

```bash List decisions
curl -s https://app.alfred.black/api/v1/vault/list/decision \
  -H "Authorization: Bearer alf_your_key_here" | jq .
```
</CodeGroup>

The Curator will have created records for the people mentioned, any tasks, decisions, and projects — all linked together.

## Step 6: Check worker status

See which agents are running:

```bash
curl -s https://app.alfred.black/api/v1/workers/status \
  -H "Authorization: Bearer alf_your_key_here"
```

## Step 7: Run a Janitor scan

Ask the Janitor to check your vault for structural issues:

```bash
curl -X POST https://app.alfred.black/api/v1/workers/janitor/scan \
  -H "Authorization: Bearer alf_your_key_here" | jq .
```

If issues are found, fix them:

```bash
curl -X POST https://app.alfred.black/api/v1/workers/janitor/fix \
  -H "Authorization: Bearer alf_your_key_here" | jq .
```
