## Prerequisites

Before you begin, make sure you have:

- An Alfred account at [app.alfred.black](https://app.alfred.black)
- An **API key** (create one in **Settings > API Keys**)
- `curl` or any HTTP client

## Step 1: Verify connectivity

Confirm your API key works by fetching the vault schema:

<CodeGroup>
```bash cURL
curl -s https://app.alfred.black/api/v1/vault/schema \
  -H "Authorization: Bearer alf_YOUR_API_KEY" | jq .
```

```javascript JavaScript
const res = await fetch('https://app.alfred.black/api/v1/vault/schema', {
  headers: { 'Authorization': 'Bearer alf_YOUR_API_KEY' }
});
console.log(await res.json());
```

```python Python
import requests

res = requests.get(
    'https://app.alfred.black/api/v1/vault/schema',
    headers={'Authorization': 'Bearer alf_YOUR_API_KEY'}
)
print(res.json())
```
</CodeGroup>

You should see a JSON response listing the 19 record types Alfred manages — from `person` and `project` to `assumption` and `decision`.

## Step 2: Explore the vault schema

See what record types are available:

```bash
curl -s https://app.alfred.black/api/v1/vault/schema \
  -H "Authorization: Bearer alf_YOUR_API_KEY" | jq .known_types
```

## Step 3: Drop your first file into the inbox

Upload a raw document (meeting notes, conversation, etc.) to the inbox. The Curator will process it automatically.

```bash
curl -X POST https://app.alfred.black/api/v1/vault/inbox \
  -H "Authorization: Bearer alf_YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "filename": "standup-2026-02-26.md",
    "content": "# Daily Standup - Feb 26\n\nAttendees: Alice, Bob, Carol\n\n## Alice\n- Finished the auth migration (project: Platform v2)\n- Blocked on API rate limiting design\n- Decision: use token bucket algorithm\n\n## Bob\n- Reviewed PR #847 for Carol\n- Starting database indexing task\n\n## Carol\n- PR #847 ready for merge\n- Meeting with Design team tomorrow about dashboard redesign"
  }'
```

## Step 4: Check what the Curator produced

Wait a few seconds for the Curator to process, then list records:

<CodeGroup>
```bash List people
curl -s https://app.alfred.black/api/v1/vault/list/person \
  -H "Authorization: Bearer alf_YOUR_API_KEY" | jq .
```

```bash List tasks
curl -s https://app.alfred.black/api/v1/vault/list/task \
  -H "Authorization: Bearer alf_YOUR_API_KEY" | jq .
```

```bash List decisions
curl -s https://app.alfred.black/api/v1/vault/list/decision \
  -H "Authorization: Bearer alf_YOUR_API_KEY" | jq .
```
</CodeGroup>

The Curator will have created records for the people mentioned, any tasks, decisions, and projects — all linked together with wikilinks.

## Step 5: Check worker status

See which Alfred workers are running:

```bash
curl -s https://app.alfred.black/api/v1/workers/status \
  -H "Authorization: Bearer alf_YOUR_API_KEY"
```

## Step 6: Run a vault health scan

Ask the Janitor to check for structural issues:

```bash
curl -X POST https://app.alfred.black/api/v1/workers/janitor/scan \
  -H "Authorization: Bearer alf_YOUR_API_KEY" | jq .
```

If issues are found, fix them:

```bash
curl -X POST https://app.alfred.black/api/v1/workers/janitor/fix \
  -H "Authorization: Bearer alf_YOUR_API_KEY" | jq .
```
