## Before you begin

You'll need:

- An **Alfred Black account** with an active subscription at [alfred.black](https://alfred.black)
- An **API key** (you'll create one in a moment)
- `curl` or any HTTP client

## Step 1: Create an API key

Go to your [Alfred Black dashboard](https://alfred.black), navigate to **Settings > API Keys**, click **Create**, give it a name, and copy the key. It starts with `alf_` and is shown only once.

<Warning>
  Copy your API key immediately — you won't be able to see it again. Store it somewhere safe.
</Warning>

## Step 2: Make sure Alfred is ready

Confirm your key works and your Alfred is healthy:

<CodeGroup>
```bash cURL
curl -s https://alfred.black/api/v1/admin/health \
  -H "Authorization: Bearer alf_your_key_here" | jq .
```

```javascript JavaScript
const res = await fetch('https://alfred.black/api/v1/admin/health', {
  headers: { 'Authorization': 'Bearer alf_your_key_here' }
});
console.log(await res.json());
```

```python Python
import requests

res = requests.get(
    'https://alfred.black/api/v1/admin/health',
    headers={'Authorization': 'Bearer alf_your_key_here'}
)
print(res.json())
```
</CodeGroup>

You should see a JSON response with health status information. If Alfred reports healthy, you're ready.

## Step 3: See what Alfred knows

Take a look at your vault schema — the types of records Alfred manages for you:

```bash
curl -s https://alfred.black/api/v1/vault/schema \
  -H "Authorization: Bearer alf_your_key_here" | jq .known_types
```

This returns all 19 record types — from `person` and `project` to `assumption` and `decision`. These are the building blocks of your world.

## Step 4: Share your first content

Hand Alfred a meeting note. You can do this from your dashboard (**Inbox** page) or via the API:

```bash
curl -X POST https://alfred.black/api/v1/vault/inbox \
  -H "Authorization: Bearer alf_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{
    "filename": "standup-2026-02-26.md",
    "content": "# Daily Standup - Feb 26\n\nAttendees: Alice, Bob, Carol\n\n## Alice\n- Finished the auth migration (project: Platform v2)\n- Blocked on API rate limiting design\n- Decision: use token bucket algorithm\n\n## Bob\n- Reviewed PR #847 for Carol\n- Starting database indexing task\n\n## Carol\n- PR #847 ready for merge\n- Meeting with Design team tomorrow about dashboard redesign"
  }'
```

## Step 5: See what the Curator prepared

Wait a few seconds for the Curator to read your notes, then see what Alfred created:

<CodeGroup>
```bash List people
curl -s https://alfred.black/api/v1/vault/list/person \
  -H "Authorization: Bearer alf_your_key_here" | jq .
```

```bash List tasks
curl -s https://alfred.black/api/v1/vault/list/task \
  -H "Authorization: Bearer alf_your_key_here" | jq .
```

```bash List decisions
curl -s https://alfred.black/api/v1/vault/list/decision \
  -H "Authorization: Bearer alf_your_key_here" | jq .
```
</CodeGroup>

The Curator will have created records for the people mentioned, any tasks, decisions, and projects — all linked together. From a single meeting note, your vault already holds a web of connected knowledge.

## Step 6: Check your specialists

See which specialists are working:

```bash
curl -s https://alfred.black/api/v1/workers/status \
  -H "Authorization: Bearer alf_your_key_here"
```

## Step 7: Ask the Janitor to look things over

Have the Janitor scan your vault for anything that needs attention:

```bash
curl -X POST https://alfred.black/api/v1/workers/janitor/scan \
  -H "Authorization: Bearer alf_your_key_here" | jq .
```

If anything needs fixing:

```bash
curl -X POST https://alfred.black/api/v1/workers/janitor/fix \
  -H "Authorization: Bearer alf_your_key_here" | jq .
```
