In this quickstart, you'll upload a raw document, watch Alfred automatically extract people, decisions, and action items, and see your vault come to life.

## Prerequisites

Before you begin, make sure you have:

- An Alfred account at [app.alfred.black](https://app.alfred.black)
- An **API key** from **Settings > API Keys**
- `curl` or any HTTP client

## Step 1: Verify your API key

Confirm your credentials work by pinging the vault schema endpoint:

<CodeGroup>
```bash cURL
curl -s https://app.alfred.black/api/v1/vault/schema \
  -H "Authorization: Bearer alf_YOUR_API_KEY" | jq .
```

```javascript JavaScript
const res = await fetch('https://app.alfred.black/api/v1/vault/schema', {
  headers: { 'Authorization': 'Bearer alf_YOUR_API_KEY' }
});
console.log(await res.json());
```

```python Python
import requests

res = requests.get(
    'https://app.alfred.black/api/v1/vault/schema',
    headers={'Authorization': 'Bearer alf_YOUR_API_KEY'}
)
print(res.json())
```
</CodeGroup>

You'll see a list of the 19 record types Alfred manages: `person`, `project`, `task`, `decision`, `conversation`, and more.

<Tip>
If you get a 401 error, double-check your API key — copy it fresh from Settings.
</Tip>

## Step 2: Upload your first document

Upload a realistic meeting notes document to your inbox. The Curator AI will process it automatically:

<CodeGroup>
```bash cURL
curl -X POST https://app.alfred.black/api/v1/vault/inbox \
  -H "Authorization: Bearer alf_YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "filename": "standup-2026-02-28.md",
    "content": "# Team Standup 2026-02-28\n\nAttendees: Sarah Chen, Marcus Johnson, Priya Patel\n\n## Updates\n- Sarah: Finished the authentication refactor. Moving to payment integration next week.\n- Marcus: Database migration is 80% complete. Blocked on schema approval from compliance.\n- Priya: Shipped the mobile onboarding flow. Downloads up 23% this week.\n\n## Decisions\n- Moving to Postgres for the analytics pipeline (Marcus will lead)\n- Hiring two senior engineers — posting goes live Monday\n\n## Action Items\n- Sarah: Draft payment integration spec by Friday\n- Marcus: Schedule compliance review meeting\n- Priya: Write post-mortem for the onboarding A/B test"
  }'
```

```javascript JavaScript
const res = await fetch('https://app.alfred.black/api/v1/vault/inbox', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer alf_YOUR_API_KEY',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    filename: 'standup-2026-02-28.md',
    content: `# Team Standup 2026-02-28

Attendees: Sarah Chen, Marcus Johnson, Priya Patel

## Updates
- Sarah: Finished the authentication refactor. Moving to payment integration next week.
- Marcus: Database migration is 80% complete. Blocked on schema approval from compliance.
- Priya: Shipped the mobile onboarding flow. Downloads up 23% this week.

## Decisions
- Moving to Postgres for the analytics pipeline (Marcus will lead)
- Hiring two senior engineers — posting goes live Monday

## Action Items
- Sarah: Draft payment integration spec by Friday
- Marcus: Schedule compliance review meeting
- Priya: Write post-mortem for the onboarding A/B test`
  })
});
console.log(await res.json());
```

```python Python
import requests
import json

content = """# Team Standup 2026-02-28

Attendees: Sarah Chen, Marcus Johnson, Priya Patel

## Updates
- Sarah: Finished the authentication refactor. Moving to payment integration next week.
- Marcus: Database migration is 80% complete. Blocked on schema approval from compliance.
- Priya: Shipped the mobile onboarding flow. Downloads up 23% this week.

## Decisions
- Moving to Postgres for the analytics pipeline (Marcus will lead)
- Hiring two senior engineers — posting goes live Monday

## Action Items
- Sarah: Draft payment integration spec by Friday
- Marcus: Schedule compliance review meeting
- Priya: Write post-mortem for the onboarding A/B test"""

res = requests.post(
    'https://app.alfred.black/api/v1/vault/inbox',
    headers={'Authorization': 'Bearer alf_YOUR_API_KEY'},
    json={
        'filename': 'standup-2026-02-28.md',
        'content': content
    }
)
print(res.json())
```
</CodeGroup>

The document is now in your inbox. By default, the Curator processes new files once per minute, but we can speed things up.

## Step 3: Trigger immediate processing

Don't wait — process your document right now:

<CodeGroup>
```bash cURL
curl -X POST https://app.alfred.black/api/v1/workers/process \
  -H "Authorization: Bearer alf_YOUR_API_KEY"
```

```javascript JavaScript
const res = await fetch('https://app.alfred.black/api/v1/workers/process', {
  method: 'POST',
  headers: { 'Authorization': 'Bearer alf_YOUR_API_KEY' }
});
console.log(await res.json());
```

```python Python
import requests

res = requests.post(
    'https://app.alfred.black/api/v1/workers/process',
    headers={'Authorization': 'Bearer alf_YOUR_API_KEY'}
)
print(res.json())
```
</CodeGroup>

The endpoint returns immediately, but processing runs in the background (typically 2-5 seconds). You'll see a response like:

```json
{
  "raw": "Processing initiated. Check /api/v1/vault/list to see extracted records."
}
```
