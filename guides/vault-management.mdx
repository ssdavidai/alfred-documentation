The vault API provides full CRUD operations on structured Obsidian records. All operations respect scope enforcement (curator/janitor/distiller rules).

## Understanding the Vault

Alfred's vault contains 19 record types organized by directory:

- **14 entity types**: person, org, project, task, event, note, location, process, account, asset, conversation, input, run, session
- **5 learning types**: assumption, decision, constraint, contradiction, synthesis

Each record is a Markdown file with YAML frontmatter.

curl -X GET https://app.alfred.black/api/v1/vault/schema \

## Reading Records

### Read a Single Record

Records are read by path (e.g., `person/Alice Johnson`):

```bash
curl -X GET https://app.alfred.black/api/v1/vault/records/person/Alice%20Johnson \
  -H "Authorization: Bearer YOUR_API_KEY"
```

**Response:**

If the underlying `alfred vault read` command returns valid JSON, it is passed through directly. Otherwise, the output is returned as `{ "raw": "<text>" }`.

```json
{ "raw": "---\ntype: person\ncreated: 2026-01-15\nname: Alice Johnson\nstatus: active\ntags:\n  - engineering\n  - backend\n---\nAlice is a senior engineer and team lead." }
```

### List Records by Type

```bash
curl -X GET https://app.alfred.black/api/v1/vault/list/project \
  -H "Authorization: Bearer YOUR_API_KEY"
```

**Response:**

If the underlying CLI returns valid JSON, it is passed through directly. Otherwise, the output is returned as `{ "raw": "<text>" }`.

### Get Vault Context

Get all recent records and relationships:

```bash
curl -X GET https://app.alfred.black/api/v1/vault/context \
  -H "Authorization: Bearer YOUR_API_KEY"
```

**Response:**

If the underlying CLI returns valid JSON, it is passed through directly. Otherwise, the output is returned as `{ "raw": "<text>" }`.

## Creating Records

### Create a Person

```bash
curl -X POST https://app.alfred.black/api/v1/vault/records \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "type": "person",
    "name": "Bob Smith",
    "fields": {
      "status": "active",
      "role": "Product Manager"
    },
    "body": "Bob leads product strategy for the platform team."
  }'
```

**Response:**

If the underlying CLI returns valid JSON, it is passed through directly. Otherwise, the output is returned as `{ "raw": "<text>" }`.

### Create a Task

```bash
curl -X POST https://app.alfred.black/api/v1/vault/records \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "type": "task",
    "name": "Migrate database to Postgres",
    "fields": {
      "status": "todo",
      "priority": "high",
      "project": "[[project/Q1 Microservices Migration]]"
    },
    "body": "Migrate the primary database from MySQL to Postgres."
  }'
```

<Note>
The only globally required fields are `type` and `created`. All other fields are optional. The `fields` object maps to `--set key=value` arguments passed to the `alfred vault create` CLI.
</Note>

## Editing Records

### Set Field Values

Replace a field value using PATCH:

```bash
curl -X PATCH https://app.alfred.black/api/v1/vault/records/task/Migrate%20database%20to%20Postgres \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "set": {
      "status": "active"
    }
  }'
```

**Response:**

If the underlying CLI returns valid JSON, it is passed through directly. Otherwise, the output is returned as `{ "raw": "<text>" }`.

### Append to List Fields

Add items to list fields (roles, tags, people, projects, etc.):

```bash
curl -X PATCH https://app.alfred.black/api/v1/vault/records/person/Alice%20Johnson \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "append": {
      "tags": "mentor"
    }
  }'
```

### Append to Body

Add content to the record body:

```bash
curl -X PATCH https://app.alfred.black/api/v1/vault/records/task/Migrate%20database%20to%20Postgres \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "body_append": "\n\n## Progress Update 2026-02-26\n\nSchema conversion complete. Starting data migration."
  }'
```

### Combined Edit

Update multiple fields at once:

```bash
curl -X PATCH https://app.alfred.black/api/v1/vault/records/task/Migrate%20database%20to%20Postgres \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "set": {
      "status": "done"
    },
    "append": {
      "tags": "completed"
    },
    "body_append": "\n\nMigration completed successfully on 2026-02-28."
  }'
```

## Moving Records

Rename a record (moves the file):

```bash
curl -X POST https://app.alfred.black/api/v1/vault/move \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "from": "project/Q1 Microservices Migration",
    "to": "project/Microservices Migration 2026"
  }'
```

**Response:**

If the underlying CLI returns valid JSON, it is passed through directly. Otherwise, the output is returned as `{ "raw": "<text>" }`.

<Warning>
Moving a record updates all wikilinks to it across the vault (if Obsidian CLI is available). Without Obsidian CLI, links become broken.
</Warning>

## Deleting Records

Delete a record:

```bash
curl -X DELETE https://app.alfred.black/api/v1/vault/records/task/Obsolete%20task \
  -H "Authorization: Bearer YOUR_API_KEY"
```

**Response:**

If the underlying CLI returns valid JSON, it is passed through directly. Otherwise, the output is returned as `{ "raw": "<text>" }`.

You can optionally pass a `scope` query parameter:

```bash
curl -X DELETE "https://app.alfred.black/api/v1/vault/records/task/Obsolete%20task?scope=janitor" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

<Warning>
Deletes are permanent. Consider updating the record's status instead (e.g., `cancelled` for tasks, `abandoned` for projects).
</Warning>

## Searching Records

### Search by Glob or Grep

Find files by pattern or content:

```bash
curl -X GET "https://app.alfred.black/api/v1/vault/search?glob=**/*Migration*" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```bash
curl -X GET "https://app.alfred.black/api/v1/vault/search?grep=high%20priority" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

**Response:**

If the underlying CLI returns valid JSON, it is passed through directly. Otherwise, the output is returned as `{ "raw": "<text>" }`.

<Tip>
Grep supports regex patterns. Use `grep=status: (todo|blocked)` to find tasks needing attention.
</Tip>

## Scope Enforcement

Operations are restricted by scope:

| Scope | Create | Edit | Move | Delete |
|-------|--------|------|------|--------|
| **curator** | all types | all types | inbox only | -- |
| **janitor** | -- | all types | -- | all types |
| **distiller** | learning types only | -- | -- | -- |
| **default** | all types | all types | all types | all types |

The tenant API runs with `scope: default` (full access) unless a scope is explicitly passed.

## Error Handling

All errors follow a consistent format:

```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Unknown vault type: invalid. Known types: person, org, ..."
  }
}
```

Error codes: `UNAUTHORIZED` (401), `VALIDATION_ERROR` (400), `NOT_FOUND` (404), `EXEC_ERROR` (500), `INTERNAL_ERROR` (500).

## Troubleshooting

### Record Not Found

Check the exact path. Record paths are case-sensitive and must match exactly:

```bash
curl -X GET https://app.alfred.black/api/v1/vault/list/person \
  -H "Authorization: Bearer YOUR_API_KEY"
```

### Invalid Type

Check the schema for valid types:

```bash
curl -X GET https://app.alfred.black/api/v1/vault/schema \
  -H "Authorization: Bearer YOUR_API_KEY"
```

### Scope Error

The tenant API has full access by default, but if you pass a `scope` parameter, operations may fail if the scope does not allow that operation on that record type.
