## Setup

Every recipe below assumes:

```bash
export ALFRED_API_BASE="https://alfred.black"
export ALFRED_API_KEY="alf_replace_me_with_your_real_key"
```

Get a key from Settings → API Keys. See [API Keys](/guides/api-keys) for details.

All endpoints respond with JSON. Pipe to `jq` for readable output.

---

## 1. List everything in your vault

A one-shot overview — record counts per type and a sample of each:

```bash
curl -s "$ALFRED_API_BASE/api/v1/vault/context" \
  -H "Authorization: Bearer $ALFRED_API_KEY" \
  | jq '.records_by_type | to_entries | map({type: .key, count: (.value | length)})'
```

---

## 2. List your active matters

```bash
curl -s "$ALFRED_API_BASE/api/v1/vault/list/matter" \
  -H "Authorization: Bearer $ALFRED_API_KEY" \
  | jq '.records[] | select(.status == "active") | {name, path}'
```

---

## 3. Search the vault

Full-text search across record bodies, optionally filtered by type:

```bash
curl -s "$ALFRED_API_BASE/api/v1/vault/search?q=lease&type=matter" \
  -H "Authorization: Bearer $ALFRED_API_KEY" \
  | jq
```

## 5. Create a vault note

```bash
curl -s -X POST "$ALFRED_API_BASE/api/v1/vault/records" \
  -H "Authorization: Bearer $ALFRED_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "type": "note",
    "name": "team-standup-2026-04-25",
    "content": "---\ntype: note\nname: team-standup-2026-04-25\ndate: 2026-04-25\n---\n# Team Standup — 25 April 2026\n\n- Sarah: still chasing the Q3 contract. ETA Friday.\n- Marcus: deployed v0.42, monitoring rollout.\n- Decision: ship the new pricing page on Tuesday."
  }' | jq
```

## 6. Drop a file into the inbox (text)

For text content (markdown, plain text, JSON), pass it as-is:

```bash
curl -s -X POST "$ALFRED_API_BASE/api/v1/vault/inbox" \
  -H "Authorization: Bearer $ALFRED_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "filename": "research-notes-ai-pricing.md",
    "content": "# AI Pricing Research\n\nKey findings from 4 hours of survey work...\n\n- Most SaaS AI tools cluster around $20-50/seat/month\n- Per-token pricing has lower psychological friction\n- Bundle pricing (seat + included credits) outperforms pure per-token by ~30% conversion"
  }' | jq
```

The file lands at `vault/inbox/research-notes-ai-pricing.md` and the Curator picks it up on the next sweep.

## 9. Trigger a chore on demand

For a chore named `weekly-team-digest`:

```bash

  -H "Authorization: Bearer $ALFRED_API_KEY" \
```

```bash
  -H "Authorization: Bearer $ALFRED_API_KEY" \
  | jq
```

## 10. Send an email via AgentMail (Sir's own inbox)

Send a new email from `alfred.<username>@mail.alfred.black`:

```bash
curl -s -X POST "$ALFRED_API_BASE/api/v1/email/send" \
  -H "Authorization: Bearer $ALFRED_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "to": ["sarah@acme.com"],
    "subject": "Q3 contract — quick sync?",
    "text": "Sarah,\n\nI saw your reply this morning. Free for a 15-minute call this week to walk through the redlines?\n\nThanks,\nAlfred (on behalf of Sir)"
  }' | jq
```

To reply in an existing thread, use `/api/v1/email/reply` with `{message_id, text, reply_all?: false, attachments?}`. See the [Email Channel](/guides/email-channel) guide.

## 12. Send an SMS from Sir's phone number

```bash
curl -s -X POST "$ALFRED_API_BASE/api/v1/phone/sms" \
  -H "Authorization: Bearer $ALFRED_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "to": "+36201234567",
    "body": "Sir asks me to remind you that the kitchen team arrives at nine. — Alfred"
  }' | jq
```

The number must be in E.164 format. The reply (if any) lands as a stream event in `sms-inbound`. The outbound also writes itself to `sms-outbound` for vault visibility.

## 13. Initiate an outbound voice call

Two modes — `tts` (one-shot text-to-speech) or `realtime` (live agent conversation):

```bash
# Mode: tts — Alfred says the message and hangs up
curl -s -X POST "$ALFRED_API_BASE/api/v1/phone/call" \
  -H "Authorization: Bearer $ALFRED_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "to": "+36201234567",
    "message": "Sir asks me to let you know he will be 15 minutes late. Thank you.",
    "mode": "tts"
  }' | jq

# Mode: realtime — Alfred opens a live conversation
curl -s -X POST "$ALFRED_API_BASE/api/v1/phone/call" \
  -H "Authorization: Bearer $ALFRED_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "to": "+36201234567",
    "message": "Sir, I am calling on his behalf about the front-desk package — could you check whether it has arrived?",
    "mode": "realtime"
  }' | jq
```

See the [Voice Channel](/guides/voice-channel) guide for what realtime mode actually does.

## 14. Create a custom webhook stream

For ingesting from a system Composio doesn't cover (a custom internal tool, a Zapier hook, an Airtable automation):

```bash
curl -s -X POST "$ALFRED_API_BASE/api/v1/streams" \
  -H "Authorization: Bearer $ALFRED_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "id": "internal-deploy-events",
    "name": "Internal Deploy Events",
    "type": "webhook",
    "source": "internal-deploy-system",
    "enabled": true,
    "webhookToken": "deploy-events-2026-04"
  }' | jq
```

The response includes the webhook URL. Configure your external system to POST events to it. The endpoint accepts JSON bodies up to 50 MB.

## 16. Authorize a sender for the email channel

By default, only authorized senders trigger a live agent reply on Sir's `alfred.<username>@mail.alfred.black` inbox. Everyone else goes to stream ingest.

```bash
# Add an authorized sender
curl -s -X POST "$ALFRED_API_BASE/api/v1/auth/senders" \
  -H "Authorization: Bearer $ALFRED_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"email": "sarah@acme.com"}' | jq

# List authorized senders
curl -s "$ALFRED_API_BASE/api/v1/auth/senders" \
  -H "Authorization: Bearer $ALFRED_API_KEY" \
  | jq

# Remove one
curl -s -X DELETE "$ALFRED_API_BASE/api/v1/auth/senders/sarah%40acme.com" \
  -H "Authorization: Bearer $ALFRED_API_KEY" \
  | jq
```

## 18. Regenerate Composio skills

After a Composio template change (new actions, edited descriptions), rebuild every connected app's `SKILL.md` so the agent picks up the changes:

```bash
curl -s -X POST "$ALFRED_API_BASE/api/v1/integrations/regenerate-skills" \
  -H "Authorization: Bearer $ALFRED_API_KEY" \
  | jq
```

No body required. Returns the list of skills regenerated.

## 20. Tail container logs

For the openclaw container, last 100 lines:

```bash
curl -s "$ALFRED_API_BASE/api/v1/admin/containers/openclaw/logs?tail=100" \
  -H "Authorization: Bearer $ALFRED_API_KEY" \
  | jq -r '.logs'
```

Replace `openclaw` with `alfred-learn`, `alfred`, `temporal`, or `ctrl-api` to switch container.
