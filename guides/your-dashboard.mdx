## What you see when you log in

The dashboard at [alfred.black](https://alfred.black) is Sir's window into everything Alfred is doing. It proxies through to your dedicated tenant via Tailscale — every page below pulls live from your own ctrl-api, never a shared backend.

## The Vault Nebula

Your homepage is the **Vault Nebula** — a real-time visualisation of your knowledge graph. Each cloud is a cluster of related vault records: people, organisations, projects, decisions, meetings, ideas, locations. The Surveyor builds the clusters by embedding every record and grouping by semantic similarity, then labelling each cluster with a generated heading.

This is not a static infographic. It updates as the Surveyor re-embeds your vault on its schedule. Hover any cluster to see the records inside; click a record to open it.

Beneath the Nebula, the dashboard surfaces:

- **Vault composition** — counts per record type (matter, task, person, org, event, instinct, observation, reflection, …)
- **Activity feed** — recent API calls, workflow runs, agent sessions
- **Worker activity charts** — what the Curator, Janitor, Distiller, and Surveyor have been doing

## The Intelligence menu — five tabs

The Intelligence menu (in the sidebar) is where Sir manages the kinetic layer — what's pending, what's in flight, and what Alfred has learned.

<Tabs>
  <Tab title="Inbox">
    What Alfred has received but not yet fully classified — uploads, fresh stream events, items needing your touch. Each card shows the source, the summary, and the destination Alfred is leaning toward. From here you can promote an inbox item directly into a matter or errand.
  </Tab>

  <Tab title="Matters">
    Your ongoing concerns. Each matter has a body (the story, written by Opus when the matter was created), a status, and links to every related person, task, decision, conversation, and event. Click a matter to see its full body and the connected records.

    Matters are where Sir's life lives in the vault. Open the kitchen renovation, the Q2 launch, your child's school year — each is its own matter with its own thread.
  </Tab>

  <Tab title="Errands">
    Discrete actionable items — tasks Alfred has surfaced from your inbox, generated from your patterns, or that Sir created directly. Each errand carries a status (`pending`, `in-progress`, `done`), an owner, optional `requires_approval`, and a body explaining what needs to happen.

    The errand detail panel shows the full body up top, then the metadata strip — built specifically so Sir can read what needs doing without scrolling.
  </Tab>

  <Tab title="Chores">
    Recurring scheduled workflows. Each chore is a real Python file deployed to your tenant — generated by Opus during onboarding, or written conversationally later. The Chores tab lets you:

    - Pause / resume / trigger any chore
    - Read the full Python source and the dependency audit (which activities it imports, what each one reads or writes, and a live data-readiness probe against the vault directories the activity depends on)
    - Edit the schedule cron expression
    - See the per-chore dry-run quarantine status (first 3 runs of every new chore are dry-run; auto-releases after 3 clean runs)
    - Delete a chore

    The dependency audit is Sir's anti-hallucination defence. If a chore claims to produce a "weekly cashflow forecast" but the activities it imports never touch any payment vault records, the audit shows that mismatch before Sir relies on the output.
  </Tab>

  <Tab title="Activity">
    Everything that has happened recently — every workflow run, every agent session, every chore execution, every API call. Filter by service, by time window, by workflow type.
  </Tab>
</Tabs>

## Other pages in the sidebar

| Page | What it does |
|---|---|
| **Apps** | The unified Connected Apps + Streams + Tools surface. Browse the 1,000+ Composio catalogue, manage OAuth connections, see per-app stream + tool controls. |
| **Inbox** | Drag and drop files (text, markdown, PDF, images, audio, video) directly to the Curator. See what's in flight. |
| **Devices** | Pair an Omi wearable, manage your other openclaw clients (mobile, desktop), revoke device tokens. |
| **Phone** | Sir's tenant phone number, authorised callers list, recent calls + SMS, Voice Bridge status. |
| **Workspace** | Browse the openclaw workspace — Sir's `MEMORY.md`, `USER.md`, `SOUL.md`, the platform `TOOLS.md`, every installed skill. |
| **Workflows** | Live Temporal workflow inspector — running, recent, failed. Drill into activity history. |
| **Logs** | Container logs with tail and filter. Switch between ctrl-api, openclaw, alfred-learn, alfred, temporal. |
| **Terminal** | An in-browser shell into Sir's tenant for ad-hoc operations (with the same auth as the rest of the dashboard). |
| **Settings** | Account, subscription, API keys, Polar customer portal link. |

## Navigation patterns

<Note>
**Read before writing.** Most pages start with a "Recent" or "Open" view. Use the search/filter UI to narrow before you mutate anything — the dashboard's operations all hit your real vault, and there is no undo on a deleted record.
</Note>

- The **back arrow** in any detail page returns you to the list with your filter intact
- **Cmd+K** (or Ctrl+K) opens the global search across vault records
- **Esc** closes any modal — file upload, fact verification, chore detail
- Charts and counters refresh automatically every few seconds; you don't need to reload

## The reconfiguring banner

When you connect (or disconnect) a Composio integration, Alfred briefly restarts the openclaw gateway to register the new tools. During that ~40-second window, the dashboard shows a **"Configuring {toolkit}…"** banner at the top. Don't try to interact with the connected app's tools until the banner clears — the gateway isn't ready yet.

The banner uses a `GET /api/v1/openclaw/ready` probe and clears once two consecutive checks return ready. There's also a hard-clear safety net at `untilMs + 5s` so a stuck banner can't block you.
