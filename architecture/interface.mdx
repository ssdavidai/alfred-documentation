## Every way you interact with Alfred

Alfred exposes multiple interfaces, but they all feed into the same pipeline: content arrives, specialists process it, your vault grows. Here's every point of contact.

## Control UI — the alfred.black dashboard

The [Alfred Black dashboard](https://alfred.black) is your window into everything Alfred is doing:

- **Home** — Command Center with stat cards (health, vault, services, devices), specialist activity charts, knowledge graph, and activity feed
- **Vault** — Browse and search records, filter by type, click through connections
- **Streams** — View and manage data pipelines feeding Alfred
- **Intuition** — See how Alfred is learning, route items awaiting your judgment, review active instincts
- **Services** — Manage services, configure which LLM model each specialist uses
- **Inbox** — Share content via file upload
- **Devices** — Approve or reject OpenClaw pairing requests

## OpenClaw TUI

SSH into your instance and use the OpenClaw command-line interface directly. This gives you agent access to your vault through the same gateway your devices use — skills, sessions, and all.

## OpenClaw Channels

Connect Alfred to messaging platforms via OpenClaw's channel system:

- **WhatsApp** — Talk to Alfred from your phone
- **iMessage** — Native Apple integration
- **Telegram** — Bot-based access
- **Discord** — Server or DM access

Each channel feeds into the same specialist pipeline. A message to Alfred on WhatsApp is processed the same way as content shared through the dashboard.

## Talk Mode — voice calls

[Alfred Talk](/features/talk-mode) gives your Alfred a voice. Alfred can call you for briefings, reminders, and check-ins — and you can call Alfred's dedicated phone number any time you need something.

Alfred speaks with the calm, warm, dry-witted composure of a proper butler — the Pennyworth persona. Everything said on a call is transcribed, structured, and added to your vault.

## Ambient Mode — continuous listening

[Ambient Mode](/features/ambient-mode) connects an Omi wearable device to Alfred. Wear it throughout your day, and every conversation — meetings, phone calls, coffee chats, passing remarks — is captured, transcribed, and processed automatically.

You mention "I should call the accountant next week" while having coffee. Alfred hears it. A task appears. No manual entry.

## Email (coming soon)

Every Alfred instance will get a dedicated email address via Agentmail.to. Forward emails to Alfred, or have services send directly to your Alfred's address.

## Device pairing

Before any device can interact with your Alfred, it must be paired and approved:

1. A device requests pairing through the OpenClaw gateway
2. The request appears as pending in your dashboard
3. You approve or reject it
4. On approval, a cryptographically random device token is generated and returned once
5. The device uses this token for all subsequent connections

Tokens can be rotated periodically (every 90 days recommended) or revoked instantly if a device is lost or compromised.

## The API

For programmatic access, Alfred provides a complete REST API with 86 endpoints across vault operations, worker control, workflow orchestration, intuition management, device management, scheduling, and instance administration.

All endpoints require Bearer token authentication and are accessible over your encrypted Tailscale network or via Cloudflare Tunnel.
