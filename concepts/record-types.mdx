## Overview

Alfred supports 25 distinct record types organized into two categories: **Entity Types** (20) for operational records and **Learning Types** (5) for extracted knowledge.

## Entity Types

Operational records representing people, projects, tasks, and other tangible entities.

<AccordionGroup>

<Accordion title="person - People and Contacts" icon="user">
**Directory**: `vault/person/`

**Purpose**: Represents individuals, contacts, team members, or stakeholders.

**Required Fields**:
- `type`: `person`
- `role`: Job title or role

**Optional Fields**:
- `org`: Organization (wikilink to org record)
- `email`: Contact email address
- `phone`: Phone number
- `tags`: Array of tags
- `related`: Related records

**Example**:
```yaml
---
type: person
role: Senior Software Engineer
org: "[[org/Acme Corp]]"
email: alice@acme.com
tags: [engineering, backend]
---
```
</Accordion>

<Accordion title="org - Organizations" icon="building">
**Directory**: `vault/org/`

**Purpose**: Represents companies, teams, departments, or organizational entities.

**Required Fields**:
- `type`: `org`

**Optional Fields**:
- `website`: Organization website
- `industry`: Industry sector
- `size`: Organization size
- `location`: Physical location
- `tags`: Array of tags

**Example**:
```yaml
---
type: org
industry: Technology
website: https://acme.com
location: "[[location/San Francisco]]"
---
```
</Accordion>

<Accordion title="project - Projects and Initiatives" icon="diagram-project">
**Directory**: `vault/project/`

**Purpose**: Represents projects, initiatives, or multi-task efforts.

**Required Fields**:
- `type`: `project`
- `status`: One of `active`, `paused`, `complete`, `archived`

**Optional Fields**:
- `owner`: Project owner (wikilink to person)
- `priority`: Priority level (e.g., high, medium, low)
- `start_date`: Project start date
- `end_date`: Project end date
- `tags`: Array of tags

**Example**:
```yaml
---
type: project
status: active
owner: "[[person/Alice Smith]]"
priority: high
start_date: 2026-01-15
tags: [q1, infrastructure]
---
```
</Accordion>

<Accordion title="task - Tasks and Action Items" icon="check">
**Directory**: `vault/task/`

**Purpose**: Represents individual tasks, action items, or todos.

**Required Fields**:
- `type`: `task`
- `status`: One of `backlog`, `active`, `done`, `dropped`

**Optional Fields**:
- `owner`: Task assignee (wikilink to person)
- `priority`: Priority level
- `project`: Parent project (wikilink)
- `due_date`: Due date
- `tags`: Array of tags

**Example**:
```yaml
---
type: task
status: active
owner: "[[person/Alice Smith]]"
project: "[[project/Q1 Launch]]"
priority: high
due_date: 2026-03-01
---
```
</Accordion>

<Accordion title="event - Events and Occurrences" icon="calendar">
**Directory**: `vault/event/`

**Purpose**: Represents one-time events, conferences, launches, or occurrences.

**Required Fields**:
- `type`: `event`

**Optional Fields**:
- `status`: One of `upcoming`, `past`
- `date`: Event date
- `location`: Event location (wikilink or string)
- `attendees`: List of attendees (wikilinks to person)
- `tags`: Array of tags

**Example**:
```yaml
---
type: event
status: upcoming
date: 2026-04-10
location: "[[location/Conference Center]]"
attendees:
  - "[[person/Alice Smith]]"
  - "[[person/Bob Jones]]"
---
```
</Accordion>

## Learning Types

Meta-records that capture latent knowledge extracted from operational records by the Distiller tool.

<AccordionGroup>

<Accordion title="assumption - Implicit Assumptions" icon="thought-bubble">
**Directory**: `vault/_learn/assumption/`

**Purpose**: Captures implicit assumptions found in operational records.

**Required Fields**:
- `type`: `assumption`

**Optional Fields**:
- `source`: Source records (wikilinks)
- `confidence`: Confidence level
- `tags`: Array of tags

**Example**:
```yaml
---
type: assumption
source:
  - "[[project/Q1 Launch]]"
  - "[[meeting/Planning Session]]"
confidence: medium
---

# Users prefer mobile-first design

Based on project discussions, we're assuming users primarily access the platform via mobile devices, though this hasn't been validated with data.
```
</Accordion>

<Accordion title="decision - Decisions and Rationale" icon="scale-balanced">
**Directory**: `vault/_learn/decision/`

**Purpose**: Captures decisions and their rationale.

**Required Fields**:
- `type`: `decision`

**Optional Fields**:
- `source`: Source records (wikilinks)
- `date`: Decision date
- `owner`: Decision maker (wikilink)
- `tags`: Array of tags

**Example**:
```yaml
---
type: decision
source:
  - "[[meeting/Architecture Review]]"
date: 2026-02-15
owner: "[[person/Alice Smith]]"
---

# Use PostgreSQL for primary database

We've decided to use PostgreSQL instead of MySQL due to better support for JSONB and full-text search, which are critical for our use case.
```
</Accordion>

<Accordion title="constraint - Constraints and Limitations" icon="triangle-exclamation">
**Directory**: `vault/_learn/constraint/`

**Purpose**: Captures constraints, limitations, or boundaries.

**Required Fields**:
- `type`: `constraint`

**Optional Fields**:
- `source`: Source records (wikilinks)
- `category`: Constraint category (technical, business, regulatory, etc.)
- `tags`: Array of tags

**Example**:

```yaml
We cannot deprecate the v1 API until Q3 2026 due to existing client contracts, which constrains our ability to refactor the backend.
```
</Accordion>

<Accordion title="contradiction - Contradictions and Conflicts" icon="circle-exclamation">
**Directory**: `vault/_learn/contradiction/`

**Purpose**: Captures contradictions or conflicts found across records.

**Required Fields**:
- `type`: `contradiction`

**Optional Fields**:
- `source`: Source records (wikilinks)
- `severity`: Severity level
- `tags`: Array of tags

**Example**:
```yaml
---
type: contradiction
source:
  - "[[goal/Launch by Q2]]"
  - "[[task/Complete Security Audit]]"
severity: high
---

# Timeline conflict between launch goal and security requirements

The Q2 launch goal conflicts with the security audit task, which requires 6 weeks and hasn't started yet. This makes the Q2 timeline infeasible.
```
</Accordion>

<Accordion title="synthesis - Syntheses and Connections" icon="diagram-venn">
**Directory**: `vault/_learn/synthesis/`

**Purpose**: Captures synthesized insights connecting multiple records.

**Required Fields**:
- `type`: `synthesis`

**Optional Fields**:

- `tags`: Array of tags

**Example**:
```yaml

```
</Accordion>

</AccordionGroup>

<Note>
Learning types are automatically extracted by the Distiller tool. You typically don't create them manually via the API.
</Note>

## Type Naming Conventions

- Type identifiers use lowercase, singular form (e.g., `person`, not `Person` or `people`)
- Directory names match type names (e.g., `person` → `vault/person/`)
- Learning types are prefixed with `_learn/` directory to distinguish them
- Record filenames use title case with spaces (e.g., `Alice Smith.md`)

<Tip>
Use the `GET /api/v1/vault/schema` endpoint to retrieve the complete, up-to-date schema programmatically. The schema is the source of truth for all validation rules.
</Tip>
