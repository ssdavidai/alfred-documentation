---
title: "Update credentials"
description: "Set or remove LLM-provider credentials in the tenant .env."
api: "PATCH /api/v1/admin/credentials"
---

Surgically updates one or more credential values in the tenant `.env`. Comments, blank lines, ordering, and unrelated keys are preserved. Pass a string to set a value, or `null` to remove it. After writing, the `openclaw` and then `alfred` containers are recreated **with `--no-deps`** so the new credentials take effect — ctrl-API itself is deliberately not restarted to avoid a 502 cascade on the Cloudflare proxy.

## Body

A flat object of `key → string | null`. Only the keys listed by [`GET /api/v1/admin/credentials`](/api-reference/credentials/list) are accepted.

<ParamField body="OPENROUTER_API_KEY" type="string | null">OpenRouter API key (200+ models).</ParamField>
<ParamField body="ANTHROPIC_API_KEY" type="string | null">Anthropic API key (direct Claude).</ParamField>
<ParamField body="OPENAI_API_KEY"    type="string | null">OpenAI API key.</ParamField>
<ParamField body="XAI_API_KEY"       type="string | null">xAI API key (Grok).</ParamField>
<ParamField body="GOOGLE_API_KEY"    type="string | null">Google API key (Gemini).</ParamField>

## Response

<ResponseField name="message"   type="string">  Confirmation including the ~30s restart hint.</ResponseField>
<ResponseField name="restarted" type="string[]">Containers being recreated. Always `["openclaw", "alfred"]`.</ResponseField>

## Errors

`400 Bad Request` — body not an object, unknown key, attempt to modify `AAS_API_KEY`, or non-string non-null value.

<RequestExample>

```bash Set one
curl -X PATCH https://alfred.black/api/v1/admin/credentials \
  -H "Authorization: Bearer alf_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{"OPENROUTER_API_KEY": "sk-or-v1-..."}'
```

```bash Set several
curl -X PATCH https://alfred.black/api/v1/admin/credentials \
  -H "Authorization: Bearer alf_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{"XAI_API_KEY": "xai-...", "ANTHROPIC_API_KEY": "sk-ant-..."}'
```

```bash Remove one
curl -X PATCH https://alfred.black/api/v1/admin/credentials \
  -H "Authorization: Bearer alf_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{"GOOGLE_API_KEY": null}'
```

</RequestExample>

<ResponseExample>

```json 200 OK
{
  "message": "Credentials updated. Services are restarting (may take ~30s).",
  "restarted": ["openclaw", "alfred"]
}
```

```json 400 Unknown key
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Unknown credential key: FOO_KEY. Known keys: OPENROUTER_API_KEY, ANTHROPIC_API_KEY, OPENAI_API_KEY, XAI_API_KEY, GOOGLE_API_KEY"
  }
}
```

```json 400 Protected
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Cannot modify protected key: AAS_API_KEY"
  }
}
```

</ResponseExample>

<Warning>
  The response returns immediately, before the containers come back up. Expect ~30s during which Alfred is unreachable. After that window, poll [`GET /api/v1/openclaw/health`](/api-reference/openclaw/health) to confirm the gateway is alive again. Avoid running this during active onboarding or chore generation.
</Warning>

<Note>
  ctrl-API and `temporal` are deliberately untouched — `--no-deps` ensures Docker only recreates `openclaw` and `alfred`. This prevents the 502 cascade that would happen if ctrl-API itself were recreated mid-request.
</Note>
