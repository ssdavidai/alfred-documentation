---
title: "Update Credentials"
description: "Set or remove API credentials for external providers."
api: "PATCH /api/v1/admin/credentials"
---

Updates one or more credential values in the tenant's `.env` file. After updating, the `alfred` and `openclaw` containers are automatically restarted so the new credentials take effect.

## Request Body

The request body is a JSON object where keys are credential names and values are either a string (to set) or `null` (to remove).

<ParamField body="OPENROUTER_API_KEY" type="string | null">
  OpenRouter API key. Routes to 200+ models.
</ParamField>

<ParamField body="ANTHROPIC_API_KEY" type="string | null">
  Anthropic API key. For direct Claude access.
</ParamField>

<ParamField body="OPENAI_API_KEY" type="string | null">
  OpenAI API key. For GPT-4 and other OpenAI models.
</ParamField>

<ParamField body="XAI_API_KEY" type="string | null">
  xAI API key. For Grok models.
</ParamField>

<ParamField body="GOOGLE_API_KEY" type="string | null">
  Google API key. For Gemini and other Google AI models.
</ParamField>

<Note>
  You can update multiple credentials in a single request. Only include the keys you want to change. Pass `null` as the value to remove a credential.
</Note>

## Response

<ResponseField name="message" type="string">
  Confirmation message.
</ResponseField>

<ResponseField name="restarted" type="string[]">
  List of containers that were restarted to pick up the new credentials. Always `["alfred", "openclaw"]`.
</ResponseField>

## Request Example

<RequestExample>

```bash cURL — Set a credential
curl -X PATCH /api/v1/admin/credentials \
  -H "Authorization: Bearer alf_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{"OPENROUTER_API_KEY": "sk-or-v1-your-key-here"}'
```

```bash cURL — Set multiple credentials
curl -X PATCH /api/v1/admin/credentials \
  -H "Authorization: Bearer alf_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{"OPENROUTER_API_KEY": "sk-or-v1-your-key", "ANTHROPIC_API_KEY": "sk-ant-your-key"}'
```

```bash cURL — Remove a credential
curl -X PATCH /api/v1/admin/credentials \
  -H "Authorization: Bearer alf_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{"ANTHROPIC_API_KEY": null}'
```

```javascript JavaScript
const response = await fetch('/api/v1/admin/credentials', {
  method: 'PATCH',
  headers: {
    'Authorization': 'Bearer alf_your_key_here',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    OPENROUTER_API_KEY: 'sk-or-v1-your-key-here'
  })
});

const data = await response.json();
console.log(data.message);    // "Credentials updated"
console.log(data.restarted);  // ["alfred", "openclaw"]
```

</RequestExample>

## Response Example

<ResponseExample>

```json 200 OK
{
  "message": "Credentials updated",
  "restarted": ["alfred", "openclaw"]
}
```

```json 400 Bad Request — Unknown key
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Unknown credential key: FOO_KEY. Known keys: OPENROUTER_API_KEY, ANTHROPIC_API_KEY, OPENAI_API_KEY, XAI_API_KEY, GOOGLE_API_KEY"
  }
}
```

```json 400 Bad Request — Protected key
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Cannot modify protected key: AAS_API_KEY"
  }
}
```

```json 400 Bad Request — Invalid value type
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Value for OPENROUTER_API_KEY must be a string or null"
  }
}
```

```json 401 Unauthorized
{
  "error": {
    "code": "UNAUTHORIZED",
    "message": "Unauthorized"
  }
}
```

</ResponseExample>

<Warning>
  Updating credentials restarts the `alfred` and `openclaw` containers. Any in-progress agent operations will be interrupted. Time credential updates for when your agents are idle.
</Warning>
