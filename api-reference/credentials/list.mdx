---
title: "List credentials"
description: "Show every known LLM-provider credential and whether it's set."
api: "GET /api/v1/admin/credentials"
---

Returns the full set of LLM-provider credentials Alfred recognises, with masked values and provenance. Used by the dashboard's Credentials panel and by any tooling that wants to verify which providers are wired up before switching an agent's model.

Values are always masked: the first 8 characters, ellipsis, then the last 4. Anything 12 characters or shorter is replaced with `****`. The internal API key (`AAS_API_KEY`) is never returned.

## Response

<ResponseField name="credentials" type="array">
  One entry per known provider key.

  <Expandable title="Credential fields">
    <ResponseField name="key"         type="string">Environment variable name.</ResponseField>
    <ResponseField name="label"       type="string">Display label (e.g. `"OpenRouter"`).</ResponseField>
    <ResponseField name="description" type="string">What the credential unlocks.</ResponseField>
    <ResponseField name="set"         type="boolean">Whether a value is currently configured.</ResponseField>
    <ResponseField name="masked"      type="string | null">Masked value, or `null` if not set.</ResponseField>
    <ResponseField name="used_by"     type="string[]">Components that consume this credential.</ResponseField>
  </Expandable>
</ResponseField>

## Known keys

| Key | Label | Used by |
|-----|-------|---------|
| `OPENROUTER_API_KEY` | OpenRouter | All OpenClaw agents, Surveyor labeler, Surveyor embedder (when configured) |
| `ANTHROPIC_API_KEY`  | Anthropic  | OpenClaw agents using `anthropic/*` models directly |
| `OPENAI_API_KEY`     | OpenAI     | OpenClaw agents using `openai/*` models |
| `XAI_API_KEY`        | xAI        | OpenClaw agents using `xai/*` Grok models |
| `GOOGLE_API_KEY`     | Google     | OpenClaw agents using `google/*` Gemini models |

<RequestExample>

```bash cURL
curl -X GET https://alfred.black/api/v1/admin/credentials \
  -H "Authorization: Bearer alf_your_key_here"
```
