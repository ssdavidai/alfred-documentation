---
title: "List Credentials"
description: "List all known API credentials and their configuration status."
api: "GET /api/v1/admin/credentials"
---

Returns every known credential slot with its current status. Values are masked for security -- only the first 8 and last 4 characters are shown. Use this to check which providers are configured before changing agent models.

## Response

<ResponseField name="credentials" type="array">
  List of credential objects, one per known provider key.

  <Expandable title="Credential object properties">
    <ResponseField name="key" type="string">
      Environment variable name (e.g., `OPENROUTER_API_KEY`, `ANTHROPIC_API_KEY`).
    </ResponseField>

    <ResponseField name="label" type="string">
      Human-readable provider name (e.g., "OpenRouter", "Anthropic").
    </ResponseField>

    <ResponseField name="description" type="string">
      What this provider is used for.
    </ResponseField>

    <ResponseField name="set" type="boolean">
      Whether this credential has a value configured.
    </ResponseField>

    <ResponseField name="masked" type="string | null">
      Masked version of the value (first 8 + last 4 characters), or `null` if not set.
    </ResponseField>

    <ResponseField name="used_by" type="string[]">
      List of components that use this credential.
    </ResponseField>
  </Expandable>
</ResponseField>

## Known credentials

| Key | Label | Used by |
|-----|-------|---------|
| `OPENROUTER_API_KEY` | OpenRouter | All OpenClaw agents, Surveyor labeler, Surveyor embedder (if configured) |
| `ANTHROPIC_API_KEY` | Anthropic | OpenClaw agents (when using `anthropic/` models directly) |
| `OPENAI_API_KEY` | OpenAI | OpenClaw agents (when using `openai/` models) |
| `XAI_API_KEY` | xAI | OpenClaw agents (when using xAI/Grok models) |
| `GOOGLE_API_KEY` | Google | OpenClaw agents (when using Google/Gemini models) |

<Note>
  The `AAS_API_KEY` (tenant internal API key) is protected and cannot be viewed or modified through this endpoint.
</Note>

## Request Example

<RequestExample>

```bash cURL
curl -X GET /api/v1/admin/credentials \
  -H "Authorization: Bearer alf_your_key_here"
```

```javascript JavaScript
const response = await fetch('/api/v1/admin/credentials', {
  method: 'GET',
  headers: {
    'Authorization': 'Bearer alf_your_key_here'
  }
});

const data = await response.json();
console.log(data.credentials);
```

</RequestExample>
