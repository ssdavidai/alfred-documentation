---
title: "List Integrations"
description: "List the third-party apps connected via Composio for this tenant."
api: "GET /api/v1/integrations"
---

## Overview

Returns the connected Composio accounts (Gmail, Google Calendar, Notion, GitHub, Slack, etc.) scoped to the current tenant via `COMPOSIO_USER_ID`. Defence-in-depth filtering ensures another tenant's connections cannot leak through if the upstream Composio API drops the `user_id` filter.

## Response

<ResponseField name="integrations" type="array">
  Array of connected accounts. Each entry has:

  - `id` (string) — Composio connected-account id.
  - `toolkit` (string) — Toolkit slug (e.g. `gmail`, `notion`, `googlecalendar`).
  - `toolkit_name` (string) — Human-friendly toolkit name.
  - `toolkit_icon` (string) — URL to the toolkit logo, may be empty.
  - `status` (string) — Composio status (`ACTIVE`, `INITIATED`, `EXPIRED`, etc.).
  - `auth_scheme` (string) — `OAUTH2`, `API_KEY`, `BEARER_TOKEN`, etc.
  - `user_id` (string) — Internal Composio member id; identifies the tenant.
  - `created_at` (string) — ISO timestamp.
</ResponseField>

<ResponseField name="count" type="number">
  Number of connections returned.
</ResponseField>

<RequestExample>

<Note>
Read isolation is enforced server-side via Composio's `user_id` query filter and re-checked client-side with `accountMatchesUserId`. If a tenant's `COMPOSIO_USER_ID` is unset or `default`, the route fails — see the per-tenant isolation gotcha in the platform notes.
</Note>
