---
title: "Ingest Conversation"
description: "Split bulk conversation into inbox files"
api: "POST /api/v1/workers/ingest"
---

Processes a bulk conversation export and splits it into individual inbox files for curator processing. Each message becomes a separate file in the vault's `inbox/` directory.

<ParamField body="dry_run" type="boolean" optional default={false}>
  If `true`, performs a dry run without writing any files. Returns what would be created.
</ParamField>

<RequestExample>

```bash Standard Ingest
curl -X POST https://app.alfred.black/api/v1/workers/ingest \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json"
```

```bash Dry Run
curl -X POST https://app.alfred.black/api/v1/workers/ingest \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "dry_run": true
  }'
```

</RequestExample>

<ResponseExample>

```json Structured Response (if alfred outputs JSON)
{
  "...": "Structure depends on alfred ingest output"
}
```

```json Raw Response (if alfred outputs text)
{
  "raw": "Ingested 47 messages"
}
```

</ResponseExample>

<Note>
  The response format depends on the `alfred ingest` command output. If the command outputs valid JSON, it is returned as-is. Otherwise, the raw text output is returned in a `raw` field.
</Note>

<ResponseField name="raw" type="string">
  Raw text output from `alfred ingest` when the output is not valid JSON. Only present when structured JSON parsing fails.
</ResponseField>
