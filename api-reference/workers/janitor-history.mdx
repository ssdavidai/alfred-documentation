---
title: "Janitor Sweep History"
description: "List the most recent Janitor sweeps."
api: "GET /api/v1/workers/janitor/history"
---

Returns the most recent Janitor sweeps in reverse chronological order, including how many files were scanned, how many issues were found, and how many were fixed or deleted in each sweep. Wraps `alfred janitor history --limit <N>`.

## Authentication

Bearer token in the `Authorization` header.

## Query parameters

<ParamField query="limit" type="string" optional default="10">
  Maximum number of sweeps to return. Passed straight through to the CLI as a string.
</ParamField>

<RequestExample>

```bash Default (10 sweeps)
curl -X GET https://alfred.black/api/v1/workers/janitor/history \
  -H "Authorization: Bearer alf_your_key_here"
```

```bash Last 25 sweeps
curl -X GET "https://alfred.black/api/v1/workers/janitor/history?limit=25" \
  -H "Authorization: Bearer alf_your_key_here"
```

</RequestExample>

<ResponseExample>

```json 200 Structured
{
  "sweeps": [
    {
      "timestamp": "2026-04-25T03:00:12Z",
      "files_scanned": 423,
      "issues_found": 7,
      "files_fixed": 0,
      "files_deleted": 0
    },
    {
      "timestamp": "2026-04-24T03:00:08Z",
      "files_scanned": 421,
      "issues_found": 12,
      "files_fixed": 10,
      "files_deleted": 0
    }
  ]
}
```

```json 200 Raw fallback
{
  "raw": "Sweep history (last 10):\nTimestamp                Issues   Fixed    Deleted\n2026-04-25T03:00:12Z     7        0        0\n2026-04-24T03:00:08Z     12       10       0"
}
```

</ResponseExample>

<ResponseField name="raw" type="string">
  Trimmed stdout when the CLI did not return JSON. Present only in the fallback case.
</ResponseField>

## When to use

- Trending vault-health metrics in the dashboard.
- Confirming a recent `janitor/fix` ran end-to-end.
- Investigating whether a spike in issues correlates with a Curator deploy.
