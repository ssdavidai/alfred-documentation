---
title: "Extract Learnings"
description: "Scan vault and extract learnings using the agent"
api: "POST /api/v1/workers/distiller/run"
---

Performs a scan for extraction candidates followed by agent-based learning extraction. The distiller agent will analyze candidate records and create new learning records (assumptions, decisions, constraints, contradictions, synthesis) for each piece of latent knowledge discovered.

<ParamField body="project" type="string" optional>
  Filter extraction to a specific project. Passed as `--project` to the underlying command. If omitted, extracts from all candidates.

  Example: `"acme-redesign"`
</ParamField>

<RequestExample>

```bash Extract All Candidates
curl -X POST https://app.alfred.black/api/v1/workers/distiller/run \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json"
```

```bash Extract from Specific Project
curl -X POST https://app.alfred.black/api/v1/workers/distiller/run \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "project": "acme-redesign"
  }'
```

</RequestExample>

<ResponseExample>

```json Structured Response (if alfred outputs JSON)
{
  "...": "Structure depends on alfred distiller run output"
}
```

```json Raw Response (if alfred outputs text)
{
  "raw": "Scanned 423 records, found 8 candidates, extracted 18 learnings"
}
```

</ResponseExample>

<Note>
  The response format depends on the `alfred distiller run` command output. If the command outputs valid JSON, it is returned as-is. Otherwise, the raw text output is returned in a `raw` field.
</Note>

<ResponseField name="raw" type="string">
  Raw text output from `alfred distiller run` when the output is not valid JSON. Only present when structured JSON parsing fails.
</ResponseField>
