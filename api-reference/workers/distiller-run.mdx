---
title: "Extract Learnings"
description: "Run Distiller knowledge extraction"
api: "POST /api/v1/workers/distiller/run"
---

Scans your vault for extraction candidates and then runs the Distiller agent to extract knowledge. The Distiller analyzes candidate records and creates new learning records -- assumptions, decisions, constraints, contradictions, and synthesis -- for each piece of latent knowledge discovered.

<ParamField body="project" type="string" optional>
  Filter extraction to a specific project. If omitted, extracts from all candidates.

  Example: `"acme-redesign"`
</ParamField>

<RequestExample>

```bash Extract All Candidates
curl -X POST /api/v1/workers/distiller/run \
  -H "Authorization: Bearer alf_your_key_here" \
  -H "Content-Type: application/json"
```

```bash Extract from Specific Project
curl -X POST /api/v1/workers/distiller/run \
  -H "Authorization: Bearer alf_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{
    "project": "acme-redesign"
  }'
```

</RequestExample>

<ResponseExample>

```json Structured Response
{
  "records_scanned": 423,
  "candidates_found": 8,
  "learnings_extracted": 18
}
```

```json Raw Response
{
  "raw": "Scanned 423 records, found 8 candidates, extracted 18 learnings"
}
```

</ResponseExample>

<Note>
  If the response contains structured JSON, it is returned directly. Otherwise, the output is returned in a `raw` field.
</Note>

<ResponseField name="raw" type="string">
  Human-readable output when structured JSON is not available.
</ResponseField>
