---
title: "Scan for Learnings"
description: "Scan vault records for learning extraction candidates"
api: "POST /api/v1/workers/distiller/scan"
---

Performs a one-time scan of vault records to identify extraction candidates without actually extracting learnings. Returns a list of records that contain latent knowledge (assumptions, decisions, constraints, contradictions, synthesis).

<ParamField body="project" type="string" optional>
  Filter candidates to a specific project. If omitted, scans all vault records.

  Example: `"acme-redesign"`
</ParamField>

<RequestExample>

```bash Scan All Records
curl -X POST https://app.alfred.black/api/v1/workers/distiller/scan \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json"
```

```bash Scan Specific Project
curl -X POST https://app.alfred.black/api/v1/workers/distiller/scan \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "project": "acme-redesign"
  }'
```

</RequestExample>

<ResponseExample>

```json Structured Response (if alfred outputs JSON)
{
  "...": "Structure depends on alfred distiller scan output"
}
```

```json Raw Response (if alfred outputs text)
{
  "raw": "Scanned 423 records, found 8 candidates"
}
```

</ResponseExample>

<Note>
  The response format depends on the `alfred distiller scan` command output. If the command outputs valid JSON, it is returned as-is. Otherwise, the raw text output is returned in a `raw` field.
</Note>

<ResponseField name="raw" type="string">
  Raw text output from `alfred distiller scan` when the output is not valid JSON. Only present when structured JSON parsing fails.
</ResponseField>
