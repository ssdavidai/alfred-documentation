---
title: "Scan for Issues"
description: "Run a janitor structural scan"
api: "POST /api/v1/workers/janitor/scan"
---

Performs a one-time structural scan of the vault without attempting to fix any issues. Returns a report of all detected problems.

<Info>
  This is a scan-only operation. To scan and fix issues, use the [fix endpoint](/api-reference/workers/janitor-fix) instead.
</Info>

<RequestExample>

```bash cURL
curl -X POST https://app.alfred.black/api/v1/workers/janitor/scan \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json"
```

</RequestExample>

<ResponseExample>

```json Structured Response (if alfred outputs JSON)
{
  "...": "Structure depends on alfred janitor scan output"
}
```

```json Raw Response (if alfred outputs text)
{
  "raw": "Scanned 423 files, found 12 issues"
}
```

</ResponseExample>

<Note>
  The response format depends on the `alfred janitor scan` command output. If the command outputs valid JSON, it is returned as-is. Otherwise, the raw text output is returned in a `raw` field.
</Note>

<ResponseField name="raw" type="string">
  Raw text output from `alfred janitor scan` when the output is not valid JSON. Only present when structured JSON parsing fails.
</ResponseField>
