---
title: "Scan for Issues"
description: "Run a Janitor vault health scan"
api: "POST /api/v1/workers/janitor/scan"
---

Runs a one-time structural health scan of your vault. The Janitor checks for broken links, invalid frontmatter, orphaned records, and other structural issues. This is a read-only operation that reports problems without modifying any files.

<Info>
  This is a scan-only operation. To scan and fix issues automatically, use the [fix endpoint](/api-reference/workers/janitor-fix) instead.
</Info>

<RequestExample>

```bash cURL
curl -X POST /api/v1/workers/janitor/scan \
  -H "Authorization: Bearer alf_your_key_here" \
  -H "Content-Type: application/json"
```

</RequestExample>

<ResponseExample>

```json Structured Response
{
  "files_scanned": 423,
  "issues_found": 12
}
```

```json Raw Response
{
  "raw": "Scanned 423 files, found 12 issues"
}
```

</ResponseExample>

<Note>
  If the response contains structured JSON, it is returned directly. Otherwise, the output is returned in a `raw` field.
</Note>

<ResponseField name="raw" type="string">
  Human-readable output when structured JSON is not available.
</ResponseField>
