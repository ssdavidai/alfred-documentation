---
title: "Janitor Scan"
description: "Read-only structural sweep of the vault."
api: "POST /api/v1/workers/janitor/scan"
---

Triggers a one-shot Janitor scan (`alfred janitor scan`). The scanner is structural-only — no LLM agent is invoked, no files are modified. It walks the vault, validates frontmatter, checks every wiki link, and lists every issue it finds.

For an actual repair pass, use [`janitor/fix`](/api-reference/workers/janitor-fix).

## Authentication

Bearer token in the `Authorization` header.

<RequestExample>

```bash cURL
curl -X POST https://alfred.black/api/v1/workers/janitor/scan \
  -H "Authorization: Bearer alf_your_key_here" \
  -H "Content-Type: application/json"
```

</RequestExample>

<ResponseExample>

```json 200 Structured
{
  "files_scanned": 423,
  "issues_found": 7,
  "issues": [
    {"file": "matters/acme-redesign.md", "type": "broken-link", "detail": "[[unknown-person]] not found"},
    {"file": "tasks/ship-mvp.md", "type": "missing-frontmatter", "detail": "type field absent"}
  ]
}
```

```json 200 Raw fallback
{
  "raw": "Scanned 423 files. Found 7 issues:\n  matters/acme-redesign.md: broken link [[unknown-person]]\n  tasks/ship-mvp.md: missing 'type' frontmatter\n  ..."
}
```

</ResponseExample>

<ResponseField name="raw" type="string">
  Trimmed stdout when the CLI did not return JSON. Present only in the fallback case.
</ResponseField>

## When to use

- Quick health check after a large bulk import or vault restore.
- Auditing what Janitor would touch before allowing it to run a repair.
- Periodic CI-style sweep on a schedule from outside the tenant.

<Note>
  Scans are cheap and side-effect-free. Run as often as you like — there's no LLM cost.
</Note>
