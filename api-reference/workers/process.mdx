---
title: "Process Inbox"
description: "Batch-process inbox files"
api: "POST /api/v1/workers/process"
---

Processes inbox files in batches using the curator. This is a one-time batch operation, distinct from the continuous daemon watcher.

<ParamField body="limit" type="number" optional>
  Maximum number of inbox files to process. If omitted, processes all files.
</ParamField>

<ParamField body="dry_run" type="boolean" optional default={false}>
  If `true`, performs a dry run without invoking the agent. Returns what would be processed.
</ParamField>

<ParamField body="jobs" type="number" optional>
  Number of parallel processing jobs.
</ParamField>

<RequestExample>

```bash Process All Files
curl -X POST https://app.alfred.black/api/v1/workers/process \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json"
```

```bash Process Limited Files
curl -X POST https://app.alfred.black/api/v1/workers/process \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "limit": 10,
    "jobs": 3
  }'
```

```bash Dry Run
curl -X POST https://app.alfred.black/api/v1/workers/process \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "dry_run": true
  }'
```

</RequestExample>

<ResponseExample>

```json Structured Response (if alfred outputs JSON)
{
  "...": "Structure depends on alfred process output"
}
```

```json Raw Response (if alfred outputs text)
{
  "raw": "Processed 10 inbox files"
}
```

</ResponseExample>

<Note>
  The response format depends on the `alfred process` command output. If the command outputs valid JSON, it is returned as-is. Otherwise, the raw text output is returned in a `raw` field.
</Note>

<ResponseField name="raw" type="string">
  Raw text output from `alfred process` when the output is not valid JSON. Only present when structured JSON parsing fails.
</ResponseField>
