---
title: "Worker Status"
description: "Overall daemon status across Curator, Janitor, Distiller, and Surveyor."
api: "GET /api/v1/workers/status"
---

Reports the current state of every worker daemon running inside the `alfred` container. This is the equivalent of running `alfred status` on the tenant VPS and is the fastest way to confirm that Sir's vault automations are alive.

The four workers cover distinct responsibilities:

- **Curator** — watches `vault/inbox/` and turns raw notes into structured records.
- **Janitor** — sweeps the vault for structural problems (broken links, bad frontmatter, orphans).
- **Distiller** — extracts latent knowledge (assumptions, decisions, constraints, contradictions, syntheses) from existing records.
- **Surveyor** — embeds and clusters records for semantic search and entity linking.

The endpoint shells out to `alfred status` and returns its raw output unparsed, so the response shape mirrors whatever the CLI prints today.

## Authentication

Bearer token in the `Authorization` header. See [Authentication](/api-reference/authentication).

<RequestExample>

```bash cURL
curl -X GET https://alfred.black/api/v1/workers/status \
  -H "Authorization: Bearer alf_your_key_here"
```

</RequestExample>

<ResponseExample>

```json 200
{
  "raw": "Daemon: running (pid 142, uptime 4h12m)\n\nCurator:\n  Backend: anthropic\n  Processed files: 1284\n\nJanitor:\n  Last sweep: 2026-04-25T03:00:12Z\n  Issues outstanding: 7\n\nDistiller:\n  Last run: 2026-04-24T21:00:00Z\n  Learnings extracted (lifetime): 312\n\nSurveyor:\n  Embeddings: 4108\n  Clusters: 47\n  Entity linking: 91% resolved"
}
```

</ResponseExample>

<ResponseField name="raw" type="string">
  Trimmed stdout of `alfred status`. Multi-line, human-readable. Parse only if absolutely needed; structure may evolve with worker upgrades.
</ResponseField>

## When to use

- Health checks after a deploy or container restart.
- Sanity-check before triggering a manual `process` or `janitor/fix` run.
- Surfacing daemon liveness in the dashboard's diagnostics panel.

<Note>
  This endpoint does not return per-worker structured fields. For that, call the dedicated `janitor/status` and `distiller/status` endpoints — they emit JSON when the underlying CLI does.
</Note>
