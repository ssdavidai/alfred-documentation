---
title: "Move Record"
description: "Move or rename a vault record. When Obsidian is running, wikilinks across the vault are updated automatically."
api: "POST /api/v1/vault/move"
---

## When to use

Rename a file (`person/Alice.md` -> `person/Alice Chen.md`) or relocate it across types/directories (`note/idea-x.md` -> `project/idea-x.md`). Both endpoints of the move are nudged for Plane forward-sync, so a rename of a `task/*` or `matter/*` record propagates to Plane within seconds.

If the Obsidian CLI is available on the tenant host, the move uses Obsidian which **automatically rewrites every wikilink across the vault** that referenced the old path. Without Obsidian, the move is a plain `rename(2)` — no wikilink updates.

## Body Parameters

<ParamField body="from" type="string" required>
  Current vault-relative path of the record (e.g., `person/Alice.md`).
</ParamField>

<ParamField body="to" type="string" required>
  New vault-relative path. Must end in `.md`. Must NOT already exist — overwriting is rejected.
</ParamField>

## Response

Response shape comes from `alfred vault move` and is JSON-passed through:

<ResponseField name="from" type="string">
  Original path.
</ResponseField>

<ResponseField name="to" type="string">
  New path.
</ResponseField>

<ResponseField name="wikilinks_updated" type="boolean">
  Present and `true` only when the move was performed via the Obsidian CLI (which rewrites links vault-wide). Absent on the filesystem fallback — meaning any `[[old/path]]` references in other records still point at the now-missing path.
</ResponseField>

If the alfred CLI produces non-JSON output, the response is wrapped as `{ "raw": "<stdout>" }`.

## Request Example

<RequestExample>

```bash cURL
curl -X POST https://alfred.black/api/v1/vault/move \
  -H "Authorization: Bearer $AAS_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "from": "person/Alice.md",
    "to": "person/Alice Chen.md"
  }'
```

```javascript JavaScript
const response = await fetch('https://alfred.black/api/v1/vault/move', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${process.env.AAS_API_KEY}`,
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    from: 'note/temporary-idea.md',
    to: 'project/temporary-idea.md',
  }),
});
const result = await response.json();
if (!result.wikilinks_updated) {
  console.warn('Obsidian unavailable — wikilinks may now dangle');
}
```

</RequestExample>

## Response Example

<ResponseExample>

```json 200 OK (Obsidian available)
{
  "from": "person/Alice.md",
  "to": "person/Alice Chen.md",
  "wikilinks_updated": true
}
```

```json 200 OK (filesystem fallback)
{
  "from": "person/Alice.md",
  "to": "person/Alice Chen.md"
}
```

```json 400 Bad Request
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "from and to are required"
  }
}
```

```json 500 Internal Server Error (destination exists)
{
  "error": {
    "code": "EXEC_ERROR",
    "message": "alfred failed (1): {\"error\": \"Destination already exists: person/Alice Chen.md\"}"
  }
}
```

```json 500 Internal Server Error (source missing)
{
  "error": {
    "code": "EXEC_ERROR",
    "message": "alfred failed (1): {\"error\": \"Source not found: person/Alice.md\"}"
  }
}
```

</ResponseExample>

## Tips

<Warning>
**Wikilink updates depend on Obsidian.** Production Alfred tenants typically have Obsidian running, but if it's down (unlikely in normal operation), a move is a pure filesystem rename and any `[[old/path]]` references elsewhere will dangle. Check `wikilinks_updated` in the response if your workflow depends on link integrity.
</Warning>

<Note>
This endpoint uses `POST /api/v1/vault/move` (not `PUT` or `PATCH`) because it operates on two paths at once — the request doesn't fit a single-resource verb.
</Note>

<Note>
A successful move nudges Plane for both the `from` and `to` paths, so renames of `matter/*` or `task/*` records reflect in Plane's title within a few seconds.
</Note>
