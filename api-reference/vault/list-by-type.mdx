---
title: "List Records by Type"
description: "List all records of a specific type"
api: "GET /api/v1/vault/list/{type}"
---

## Overview

Returns a list of all records for a given type. The type must be one of the 19 types hardcoded in the API. The endpoint runs `alfred vault list <type>` and returns the CLI output.

## Path Parameters

<ParamField path="type" type="string" required>
  The record type to list. Must be one of:
  - Entity types: `person`, `org`, `project`, `task`, `event`, `note`, `location`, `process`, `account`, `asset`, `conversation`, `input`, `run`, `session`
  - Learning types: `assumption`, `decision`, `constraint`, `contradiction`, `synthesis`
</ParamField>

## Response

The response structure depends on the `alfred vault list` CLI output. If the CLI returns valid JSON, it is passed through directly. Otherwise, the raw text output is wrapped in a `{ "raw": "..." }` object.

<ResponseField name="raw" type="string">
  If the CLI output is not valid JSON, the raw text output is returned in this field.
</ResponseField>

## Request Example

<RequestExample>

```bash cURL
curl -X GET \
  https://app.alfred.black/api/v1/vault/list/person \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```javascript JavaScript
const type = 'person';

const response = await fetch(`https://app.alfred.black/api/v1/vault/list/${type}`, {
  method: 'GET',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY'
  }
});

const data = await response.json();
console.log(data);
```

</RequestExample>

## Response Example

<ResponseExample>

```json 200 OK (raw text fallback)
{
  "raw": "person/Alice Chen.md\nperson/Bob Martinez.md\nperson/Carol Zhang.md"
}
```

```json 400 Bad Request
{
  "error": {
    "code": "VALIDATION_ERROR",

```

```json 401 Unauthorized
{
  "error": {
    "code": "UNAUTHORIZED",
    "message": "Unauthorized"
  }

```

</ResponseExample>
