---
title: "List Records by Type"
description: "List all records of a specific type"
api: "GET /api/v1/vault/list/{type}"
---

## Overview

Returns a list of all records for a given type. The type must be one of the 25 known entity or learning types.

## Path Parameters

<ParamField path="type" type="string" required>
  The record type to list. Must be one of:
  - Entity types: `person`, `org`, `project`, `task`, `meeting`, `document`, `resource`, `tool`, `skill`, `goal`, `milestone`, `event`, `location`, `asset`, `product`, `service`, `process`, `policy`, `standard`, `metric`
  - Learning types: `assumption`, `decision`, `constraint`, `contradiction`, `synthesis`
</ParamField>

## Response

<ResponseField name="type" type="string">
  The requested record type
</ResponseField>

<ResponseField name="records" type="array">
  Array of record objects
  <ResponseField name="path" type="string">
    Relative path to the record file
  </ResponseField>
  <ResponseField name="name" type="string">
    Display name of the record (from frontmatter or filename)
  </ResponseField>
  <ResponseField name="status" type="string">
    Current status (varies by type, e.g., `active`, `completed`, `archived`)
  </ResponseField>
  <ResponseField name="created" type="string">
    ISO 8601 creation timestamp
  </ResponseField>
</ResponseField>

<ResponseField name="count" type="number">
  Number of records returned
</ResponseField>

## Request Example

<RequestExample>

```bash cURL
curl -X GET \
  https://alfred-acme.tailnet.ts.net:3100/api/v1/vault/list/person \
  -H "Authorization: Bearer YOUR_API_KEY"
```
