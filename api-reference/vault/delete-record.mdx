---
title: "Delete Record"
description: "Delete a vault record"
api: "DELETE /api/v1/vault/records/{path}"
---

## Overview

Permanently deletes a vault record. The endpoint runs `alfred vault delete <path>` and returns the CLI output. Use with caution -- this operation cannot be undone unless you have vault backups.

## Path Parameters

<ParamField path="path" type="string" required>
  Relative path to the record (e.g., `person/Alice.md`). URL-encode if the path contains spaces or special characters.
</ParamField>

## Query Parameters

<ParamField query="scope" type="string" optional>
  Operation scope for authorization (`curator`, `janitor`, `distiller`). Defaults to no scope restrictions.
</ParamField>

## Response

The response structure depends on the `alfred vault delete` CLI output. If the CLI returns valid JSON, it is passed through directly. Otherwise, the raw text output is wrapped in a `{ "raw": "..." }` object.

<ResponseField name="raw" type="string">
  If the CLI output is not valid JSON, the raw text output is returned in this field.
</ResponseField>

## Request Example

<RequestExample>

```bash cURL
curl -X DELETE \
  "https://app.alfred.black/api/v1/vault/records/person/Alice%20Chen.md?scope=janitor" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```javascript JavaScript
const recordPath = 'person/Alice Chen.md';
const encodedPath = encodeURIComponent(recordPath);

const response = await fetch(`https://app.alfred.black/api/v1/vault/records/${encodedPath}?scope=janitor`, {
  method: 'DELETE',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY'
  }
});

const result = await response.json();
console.log(result);
```

</RequestExample>
