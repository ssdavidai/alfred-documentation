---
title: "List Inbox Files"
description: "List all files currently in the vault inbox"
api: "GET /api/v1/vault/inbox"
---

## Overview

Returns a list of all files currently in the vault's inbox directory. These are files waiting to be processed by Alfred's Curator daemon.

## Response

<ResponseField name="files" type="array">
  Array of filenames in the inbox
  <ResponseField name="file" type="string">
    Name of each file (e.g., `meeting-notes-2026-02-26.md`)
  </ResponseField>
</ResponseField>

<ResponseField name="count" type="number">
  Number of files in the inbox
</ResponseField>

## Request Example

<RequestExample>

```bash cURL
curl -X GET \
  https://alfred-acme.tailnet.ts.net:3100/api/v1/vault/inbox \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```javascript JavaScript
const response = await fetch('https://alfred-acme.tailnet.ts.net:3100/api/v1/vault/inbox', {
  method: 'GET',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY'
  }
});

const inbox = await response.json();
console.log(inbox);
```

</RequestExample>

## Response Example

<ResponseExample>

```json 200 OK
{
  "files": [
    "meeting-notes-2026-02-26.md",
    "project-idea.md",
    "contact-info.txt",
    "resume-alice-chen.pdf",
    "slack-export-2026-02.json"
  ],
  "count": 5
}
```

```json 200 OK (Empty Inbox)
{
  "files": [],
  "count": 0
}
```

</ResponseExample>
