---

## Overview

Returns the vault schema as hardcoded in the API server. This includes all 19 known record types, status values by type, list fields, required fields, name fields, and directory mappings. This endpoint does not call the Alfred CLI -- it returns the API's own static schema data.

## Response

<ResponseField name="known_types" type="array">
  List of all 19 valid record types (14 entity types + 5 learning types)
</ResponseField>

<ResponseField name="status_by_type" type="object">
  Map of record type to allowed status values. Types without status constraints (e.g., `event`) are omitted.
</ResponseField>

<ResponseField name="list_fields" type="array">
  Array of field names that are treated as multi-value lists (arrays) across all record types.
</ResponseField>

<ResponseField name="required_fields" type="array">
  Array of globally required frontmatter fields (applies to all types).
</ResponseField>

<ResponseField name="name_field_by_type" type="object">
  Map of record type to the field used as the display name. Only types that override the default (`name`) are included.
</ResponseField>

<ResponseField name="type_directory" type="object">
  Map of record type to its subdirectory in the vault. Types not listed here (e.g., `session`, `input`) can be stored flexibly.
</ResponseField>

## Request Example

<RequestExample>

```bash cURL
curl -X GET \
  https://app.alfred.black/api/v1/vault/schema \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```javascript JavaScript
const response = await fetch('https://app.alfred.black/api/v1/vault/schema', {
  method: 'GET',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY'
  }
});

const schema = await response.json();
console.log(schema);
```

</RequestExample>
