---
title: "Get Vault Schema"
description: "Returns the complete vault schema definition."
api: "GET /api/v1/vault/schema"
---

## Overview

Returns the complete vault schema including all record types, valid status values, list fields, required fields, and name field overrides. Use this to understand what record types are available and how they're structured.

## Response

<ResponseField name="known_types" type="array">
  All 19 valid record types.
</ResponseField>

<ResponseField name="status_by_type" type="object">
  Map of record type to allowed status values. Types without status constraints (e.g., `event`) are omitted.
</ResponseField>

<ResponseField name="list_fields" type="array">
  Field names that accept multiple values as arrays.
</ResponseField>

<ResponseField name="required_fields" type="array">
  Globally required fields (applies to all types).
</ResponseField>

<ResponseField name="name_field_by_type" type="object">
  Types that use a different title field instead of `name`. Only `conversation` and `input` (which use `subject`) are listed.
</ResponseField>

<ResponseField name="type_directory" type="object">
  Map of record type to its directory in the vault.
</ResponseField>

## Request Example

<RequestExample>

```bash cURL
curl -X GET /api/v1/vault/schema \
  -H "Authorization: Bearer alf_your_key_here"
```

```javascript JavaScript
const response = await fetch('/api/v1/vault/schema', {
  method: 'GET',
  headers: {
    'Authorization': 'Bearer alf_your_key_here'
  }
});

const schema = await response.json();
console.log(schema);
```

</RequestExample>
