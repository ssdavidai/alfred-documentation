title: "Get Vault Schema"

---

## Overview

Returns the full vault schema including all known record types, status values, field definitions, and directory structure. This endpoint is useful for validating record creation and understanding the vault's type system.

## Response

<ResponseField name="known_types" type="array">
  List of all 25 valid record types (20 entity types + 5 learning types)
</ResponseField>

<ResponseField name="learn_types" type="array">
  Subset of learning/knowledge types: `assumption`, `decision`, `constraint`, `contradiction`, `synthesis`
</ResponseField>

<ResponseField name="status_by_type" type="object">
  Map of record type to allowed status values
  <ResponseField name="person" type="array">
    e.g., `["active", "inactive", "archived"]`
  </ResponseField>
  <ResponseField name="project" type="array">
    e.g., `["planning", "active", "on_hold", "completed", "cancelled"]`
  </ResponseField>
</ResponseField>

<ResponseField name="list_fields" type="object">
  Map of record type to fields that contain arrays
  <ResponseField name="person" type="array">
    e.g., `["skills", "projects", "orgs"]`
  </ResponseField>
</ResponseField>

<ResponseField name="required_fields" type="object">
  Map of record type to required frontmatter fields
  <ResponseField name="person" type="array">
    e.g., `["type", "status"]`
  </ResponseField>
</ResponseField>

<ResponseField name="name_field_by_type" type="object">
  Map of record type to the field used as the display name
  <ResponseField name="person" type="string">
    e.g., `"name"`
  </ResponseField>
</ResponseField>

<ResponseField name="type_directory" type="object">
  Map of record type to its subdirectory in the vault
  <ResponseField name="person" type="string">
    e.g., `"person"`
  </ResponseField>
  <ResponseField name="assumption" type="string">
    e.g., `"_learn/assumption"`
  </ResponseField>
</ResponseField>

## Request Example

<RequestExample>

```bash cURL
curl -X GET \
  https://alfred-acme.tailnet.ts.net:3100/api/v1/vault/schema \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```javascript JavaScript
const response = await fetch('https://alfred-acme.tailnet.ts.net:3100/api/v1/vault/schema', {
  method: 'GET',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY'
  }
});

const schema = await response.json();
console.log(schema);
```

</RequestExample>
