---
title: "Read Record"
description: "Read a specific vault record by path"
api: "GET /api/v1/vault/records/{path}"
---

## Overview

Returns the contents of a vault record. The endpoint runs `alfred vault read <path>` and returns the CLI output. If the CLI returns valid JSON (e.g., parsed frontmatter and body), it is passed through directly. Otherwise, the raw text output is wrapped in a `{ "raw": "..." }` object.

## Path Parameters

<ParamField path="path" type="string" required>
  Relative path to the record (e.g., `person/Alice.md`, `project/ML Pipeline.md`). URL-encode if the path contains spaces or special characters.
</ParamField>

## Response

The response structure depends on the `alfred vault read` CLI output. If the CLI returns valid JSON, it is passed through directly. Otherwise, the raw text output is wrapped in a `{ "raw": "..." }` object.

<ResponseField name="raw" type="string">
  If the CLI output is not valid JSON, the raw text output is returned in this field.
</ResponseField>

## Request Example

<RequestExample>

```bash cURL
curl -X GET \
  https://app.alfred.black/api/v1/vault/records/person/Alice%20Chen.md \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```javascript JavaScript
const recordPath = 'person/Alice Chen.md';
const encodedPath = encodeURIComponent(recordPath);

const response = await fetch(`https://app.alfred.black/api/v1/vault/records/${encodedPath}`, {
  method: 'GET',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY'
  }
});

const record = await response.json();
console.log(record);
```

</RequestExample>

## Response Example

<ResponseExample>

```json 200 OK (raw text fallback)
}
```
