title: "Search Vault"
description: "Search vault records by file pattern or content"
api: "GET /api/v1/vault/search"
---

## Overview

Search for vault records using file glob patterns or full-text content search. Both search methods can be combined. The endpoint runs `alfred vault search` and returns the CLI output.

## Query Parameters

<ParamField query="glob" type="string" optional>
  File glob pattern to match (e.g., `person/*.md`, `**/*alice*.md`)
</ParamField>

<ParamField query="grep" type="string" optional>
  Text to search for in file contents (case-insensitive)
</ParamField>

## Response

The response structure depends on the `alfred vault search` CLI output. If the CLI returns valid JSON, it is passed through directly. Otherwise, the raw text output is wrapped in a `{ "raw": "..." }` object.

<ResponseField name="raw" type="string">
  If the CLI output is not valid JSON, the raw text output is returned in this field.
</ResponseField>

## Request Example

<RequestExample>

```bash cURL
curl -X GET \
  "https://app.alfred.black/api/v1/vault/search?grep=machine%20learning&glob=project/*.md" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```javascript JavaScript
const params = new URLSearchParams({
  grep: 'machine learning',
  glob: 'project/*.md'
});

const response = await fetch(`https://app.alfred.black/api/v1/vault/search?${params}`, {
  method: 'GET',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY'
  }
});

const results = await response.json();
console.log(results);
```

</RequestExample>
