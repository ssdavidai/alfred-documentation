---
title: "Search Vault"
description: "Search vault records by file pattern or content"
api: "GET /api/v1/vault/search"
---

## Overview

Search for vault records using file glob patterns or full-text content search. Both search methods can be combined.

## Query Parameters

<ParamField query="glob" type="string" optional>
  File glob pattern to match (e.g., `person/*.md`, `**/*alice*.md`)
</ParamField>

<ParamField query="grep" type="string" optional>
  Text to search for in file contents (case-insensitive)
</ParamField>

## Response

<ResponseField name="results" type="array">
  Array of matching file paths
  <ResponseField name="path" type="string">
    Relative path to the record (e.g., `person/Alice.md`)
  </ResponseField>
</ResponseField>

<ResponseField name="count" type="number">
  Number of matching results
</ResponseField>

## Request Example

<RequestExample>

```bash cURL
curl -X GET \
  "https://alfred-acme.tailnet.ts.net:3100/api/v1/vault/search?grep=machine%20learning&glob=project/*.md" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```javascript JavaScript
const params = new URLSearchParams({
  grep: 'machine learning',
  glob: 'project/*.md'
});

const response = await fetch(`https://alfred-acme.tailnet.ts.net:3100/api/v1/vault/search?${params}`, {
  method: 'GET',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY'
  }
});

const results = await response.json();
console.log(results);
```

</RequestExample>

## Response Example

<ResponseExample>

```json 200 OK
{
  "results": [
    "project/ML Pipeline Redesign.md",
    "project/Customer Segmentation Model.md",
    "project/NLP Research Initiative.md"

```


</ResponseExample>
