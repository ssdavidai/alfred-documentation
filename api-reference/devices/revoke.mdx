---
title: "Revoke Device Access"
description: "Revoke specific role access for a paired device"
api: "POST /api/v1/devices/{deviceId}/revoke"
---

Revokes specific role-based access for a paired device without fully removing it. The device remains paired but loses access to operations requiring the revoked role.

<Note>
  This endpoint uses `deviceId` in the URL path (not `requestId`). This is the identifier of an already-paired device.
</Note>

<ParamField path="deviceId" type="string" required>
  The unique identifier of the paired device whose access should be revoked.
</ParamField>

<ParamField body="role" type="string" required>
  The role to revoke from the device. Valid values: `agent`, `user`, `admin`. This field is required and will return a validation error if omitted.
</ParamField>

<RequestExample>

```bash cURL
curl -X POST https://app.alfred.black/api/v1/devices/dev_x9y8z7w6v5/revoke \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "role": "admin"
  }'
```

</RequestExample>

<ResponseExample>

```json Response
{
  "message": "Access revoked"
}
```

```json Validation Error
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "role is required"
  }
}
```

</ResponseExample>

<ResponseField name="message" type="string">
  Human-readable status message. Contains the stdout from `openclaw devices revoke`, or `"Access revoked"` if stdout is empty.
</ResponseField>
