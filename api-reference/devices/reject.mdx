---
title: "Reject a device"
description: "Decline a pending pairing request."
api: "POST /api/v1/devices/{requestId}/reject"
---

Rejects a pending pairing request. The request is removed from the queue and no credentials are issued. Use this for unrecognised requests or for cleaning up stale pending entries.

## Path parameters

<ParamField path="requestId" type="string" required>
  The `request_id` from `GET /api/v1/devices` (under `pending`).
</ParamField>

## Response

<ResponseField name="message" type="string">
  Confirmation string from the OpenClaw CLI (e.g. `"Device rejected"`).
</ResponseField>

<RequestExample>

```bash cURL
curl -X POST https://alfred.black/api/v1/devices/req-91a3/reject \
  -H "Authorization: Bearer alf_your_key_here"
```

</RequestExample>

<ResponseExample>

```json 200 OK
{
  "message": "Device rejected"
}
```

</ResponseExample>

<Note>
  Rejection only affects the named pending request. The originating device may submit a new pairing request immediately afterwards — there is no permanent block list.
</Note>
