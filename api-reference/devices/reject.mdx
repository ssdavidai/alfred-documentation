---
title: "Reject Device"
description: "Reject a pending device pairing request"
api: "POST /api/v1/devices/{requestId}/reject"
---

Rejects a pending device pairing request, preventing the device from connecting to the OpenClaw gateway.

<Warning>
  This endpoint uses `requestId` in the URL path, not `deviceId`. The request ID is the identifier of the pending pairing request.
</Warning>

<ParamField path="requestId" type="string" required>
  The unique identifier of the pending pairing request to reject.
</ParamField>

<RequestExample>

```bash cURL
curl -X POST https://app.alfred.black/api/v1/devices/req_7k9m3p2q1/reject \
  -H "Authorization: Bearer YOUR_API_KEY"
```

</RequestExample>

<ResponseExample>

```json Response
{
  "message": "Device rejected"
}
```

</ResponseExample>

<ResponseField name="message" type="string">
  Human-readable status message confirming the request has been rejected.
</ResponseField>
