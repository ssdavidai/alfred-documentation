---
title: "Approve Device"
description: "Approve a pending device pairing request"
api: "POST /api/v1/devices/{requestId}/approve"
---

Approves a pending device pairing request, allowing the device to connect to the OpenClaw gateway.

<Warning>
  This endpoint uses `requestId` in the URL path, not `deviceId`. The request ID is the identifier of the pending pairing request, which is different from the device ID assigned after pairing.
</Warning>

<ParamField path="requestId" type="string" required>
  The unique identifier of the pending pairing request to approve.
</ParamField>

<ParamField body="latest" type="boolean" optional default={false}>
  If true, passes `--latest` flag to approve only the most recent request from this device. Prevents approval of stale requests.
</ParamField>

<RequestExample>

```bash cURL
curl -X POST https://app.alfred.black/api/v1/devices/req_7k9m3p2q1/approve \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "latest": true
  }'
```

```bash Without Options
curl -X POST https://app.alfred.black/api/v1/devices/req_7k9m3p2q1/approve \
  -H "Authorization: Bearer YOUR_API_KEY"
```

</RequestExample>

<ResponseExample>

```json Response
{
  "message": "Device approved"
}
```

</ResponseExample>

<ResponseField name="message" type="string">
  Human-readable status message confirming the device has been approved.
</ResponseField>
