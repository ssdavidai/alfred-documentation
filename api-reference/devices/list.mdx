---
title: "List devices"
description: "Show every paired device and pending pairing request."
api: "GET /api/v1/devices"
---

Returns the complete inventory of devices known to your Alfred instance. This includes both **paired devices** (anything from your phone to a CLI session that has been admitted) and **pending requests** (new pairings waiting for your approval).

Sir's dashboard polls this endpoint to render the Devices page; you can call it directly when you need a JSON view.

## How it works

The ctrl-API shells into the OpenClaw container and runs `openclaw devices list --json`. The response is whatever OpenClaw produces, passed through unchanged.

## Response

<ResponseField name="devices" type="array">
  Every paired device. Each entry typically includes a `device_id`, a human label, the role (e.g. `user`, `agent`), the scope, and the last-seen timestamp.
</ResponseField>

<ResponseField name="pending" type="array">
  Pairing requests that have not yet been approved or rejected. Each entry carries a `request_id` you can pass to the approve or reject endpoints.
</ResponseField>

<Note>
  The exact shape comes from the OpenClaw CLI. Newer versions may add fields; treat unknown keys as forward-compatible.
</Note>

<RequestExample>

```bash cURL
curl -X GET https://alfred.black/api/v1/devices \
  -H "Authorization: Bearer alf_your_key_here"
```

</RequestExample>

## When to use

- Auditing which devices currently hold credentials to your Alfred.
- Building a custom approval flow outside the dashboard.
- Spotting stale pairings before rotating tokens.
