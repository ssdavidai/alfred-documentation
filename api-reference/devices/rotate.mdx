---
title: "Rotate a device token"
description: "Issue a fresh token for one role on a paired device."
api: "POST /api/v1/devices/{deviceId}/rotate"
---

Rotates the token for a specific role on a paired device. The previous token is invalidated and a new one is issued. Use this on any whiff of credential exposure, or as routine hygiene every few months.

## Path parameters

<ParamField path="deviceId" type="string" required>
  The `device_id` from `GET /api/v1/devices`.
</ParamField>

## Body parameters

<ParamField body="role" type="string" required>
  The role whose token should be rotated (e.g. `user`, `agent`, `admin`).
</ParamField>

<ParamField body="scope" type="string">
  Optional new scope to apply during rotation. If omitted, the existing scope is preserved.
</ParamField>

## Response

<ResponseField name="message" type="string">
  Confirmation string from the OpenClaw CLI. May embed the new token depending on OpenClaw version — capture it immediately.
</ResponseField>

## Errors

`400 Bad Request` — returned when `role` is missing or not a string.

<RequestExample>

```bash cURL
curl -X POST https://alfred.black/api/v1/devices/phn-7f2c/rotate \
  -H "Authorization: Bearer alf_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{"role": "user"}'
```

```bash Rotate and re-scope
curl -X POST https://alfred.black/api/v1/devices/phn-7f2c/rotate \
  -H "Authorization: Bearer alf_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{"role": "user", "scope": "vault:read vault:write"}'
```

</RequestExample>

<ResponseExample>

```json 200 OK
{
  "message": "Token rotated"
}
```

</ResponseExample>

<Warning>
  Sir's device must be re-onboarded with the new token before the old one expires from any client cache. If the device is offline at rotation time, expect to re-pair it from scratch.
</Warning>
