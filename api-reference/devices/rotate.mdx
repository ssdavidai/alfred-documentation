---
title: "Rotate Device Token"
description: "Rotate authentication token for a paired device"
api: "POST /api/v1/devices/{deviceId}/rotate"
---

Rotates the authentication token for a paired device. The device will need to re-authenticate with the new token. Optionally update the device scope during rotation.

<Note>
  This endpoint uses `deviceId` in the URL path (not `requestId`). This is the identifier of an already-paired device.
</Note>

<ParamField path="deviceId" type="string" required>
  The unique identifier of the paired device whose token should be rotated.
</ParamField>

<ParamField body="role" type="string" required>
  The role to assign to the device. Valid values: `agent`, `user`, `admin`. This field is required and will return a validation error if omitted.
</ParamField>

<ParamField body="scope" type="string" optional>
  Optional scope restrictions for the device token.
</ParamField>

<RequestExample>

```bash With Scope
curl -X POST https://app.alfred.black/api/v1/devices/dev_a1b2c3d4e5/rotate \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "role": "agent",
    "scope": "vault:read vault:write"
  }'
```

```bash Role Only
curl -X POST https://app.alfred.black/api/v1/devices/dev_a1b2c3d4e5/rotate \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "role": "agent"
  }'
```

</RequestExample>

<ResponseExample>

```json Response
{
  "message": "Token rotated"
}
```

```json Validation Error
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "role is required"
  }
}
```

</ResponseExample>

<ResponseField name="message" type="string">
  Human-readable status message confirming the token has been rotated.
</ResponseField>
