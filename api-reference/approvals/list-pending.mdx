---
title: "List Pending Approvals"
description: "List execution-task records waiting for Sir's approve/reject decision."
api: "GET /api/v1/approvals/pending"
---

## Overview

Walks the vault for `task` records with `requires_approval: true` and `status` in `queued` or `todo`. Returns enough metadata for the dashboard's Approval card to render the gist (description, source instinct, trust tier, confidence score) without a second fetch.

Tasks are created by `JudgmentWorkflow` when an instinct fires below the trust threshold (Tier <10 obs forced approval, 10–50 configurable, 50+ autonomous).

## Response

<ResponseField name="results" type="array">
  Each entry: `{path, name, description, status, created, tier, source_instinct, confidence_score}`. Sorted by `created` desc.
</ResponseField>

<ResponseField name="count" type="number">Total pending.</ResponseField>

## Request Example

<RequestExample>

```bash cURL
curl -H "Authorization: Bearer $AAS_API_KEY" \
  https://alfred.black/api/v1/approvals/pending
```

</RequestExample>
