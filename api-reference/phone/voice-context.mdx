---

## Overview

Returns the context bundle the Voice Bridge inlines into the OpenAI Realtime `instructions` at call start, so Sir's voice agent already knows about this morning's Slack chat, his open matters, and which third-party tools are wired up. Cached for 60 seconds in-memory to avoid re-walking the vault on every call.

## Response

<ResponseField name="memoryMd" type="string">
  Tail of `MEMORY.md` (capped at 8KB).
</ResponseField>

<ResponseField name="voiceSkill" type="string">
  Contents of `skills/alfred-voice/SKILL.md` (capped at 8KB).
</ResponseField>

<ResponseField name="openMatters" type="array">
  Up to 20 active matters from `vault/matter/`. Each `{name, summary?}`.
</ResponseField>

<ResponseField name="openTasks" type="array">
  Up to 20 active tasks from `vault/task/`. Each `{name, due?, summary?}`.
</ResponseField>

<ResponseField name="recentSessions" type="array">
  Last 20 main-agent sessions tailed from `system-openclaw-sessions.jsonl`. Each `{at, channel, summary}`.
</ResponseField>

<ResponseField name="composioToolkits" type="array">
  Each `{toolkit, actions: [{name, description}]}` extracted from each `alfred-composio-<toolkit>/SKILL.md` action table.
</ResponseField>

<ResponseField name="generatedAt" type="string">ISO timestamp.</ResponseField>
