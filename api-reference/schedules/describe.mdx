---
title: "Describe Schedule"
description: "Get detailed information about a specific schedule"
api: "GET /api/v1/schedules/{schId}"
---

Returns detailed information about a workflow schedule from `temporal schedule describe --output json`. If the output is a single JSON object, it is returned parsed. Otherwise, JSONL lines are parsed into an array.

<ParamField path="schId" type="string" required>
  The schedule ID to describe
</ParamField>

<RequestExample>
```bash cURL
curl -X GET \
  -H "Authorization: Bearer YOUR_API_KEY" \
  https://app.alfred.black/api/v1/schedules/curator-hourly
```
</RequestExample>

<ResponseExample>
```json 200
{
  "schedule": {
    "spec": {
      "cronExpressions": ["0 * * * *"]
    },
    "action": {
      "startWorkflow": {
        "workflowType": {
          "name": "CuratorWorkflow"
        },
        "taskQueue": {
          "name": "alfred-curator"
        }
      }
    },
    "state": {
      "paused": false,
      "notes": ""
    }
  },
  "info": {
    "recentActions": [],
    "nextActionTimes": ["2026-02-26T15:00:00Z"]
  }
}
```
</ResponseExample>

<Note>
  The response structure is determined by the Temporal CLI's output. The exact fields may vary across Temporal versions.
</Note>
