---
title: "List secrets"
description: "Show every credential the OpenClaw gateway has access to (values masked)."
api: "GET /api/v1/openclaw/secrets"
---

Lists secrets configured for the OpenClaw gateway — provider API keys (OpenRouter, Anthropic, OpenAI, Google, xAI), Composio credentials, AgentMail keys, and so on. Values are always masked; only the key name and a redacted preview are returned.

## How it works

ctrl-API shells into OpenClaw and runs `openclaw secrets`. JSON output is forwarded; non-JSON text is returned in a `raw` field.

<RequestExample>

```bash cURL
curl -X GET https://alfred.black/api/v1/openclaw/secrets \
  -H "Authorization: Bearer alf_your_key_here"
```

</RequestExample>

<ResponseExample>

```json Structured
{
  "secrets": [
    { "name": "OPENROUTER_API_KEY", "set": true, "preview": "sk-or-v1-***" },
    { "name": "ANTHROPIC_API_KEY",  "set": true, "preview": "sk-ant-***"   },
    { "name": "GOOGLE_API_KEY",     "set": false, "preview": null          }
  ]
}
```

```json Raw fallback
{
  "raw": "OPENROUTER_API_KEY=sk-or-v1-***\nANTHROPIC_API_KEY=sk-ant-***"
}
```

</ResponseExample>

<Warning>
  This endpoint never returns plaintext secret values, even to an authenticated request. To rotate or replace a credential, use [`PATCH /api/v1/admin/credentials`](/api-reference/credentials/update).
</Warning>
