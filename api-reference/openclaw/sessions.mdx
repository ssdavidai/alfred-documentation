---
title: "List sessions"
description: "Show every active and recently completed OpenClaw session."
api: "GET /api/v1/openclaw/sessions"
---

Lists OpenClaw sessions. A session is one conversation between a caller (Slack, the dashboard, an inbound email, an ephemeral subagent) and the gateway. Useful for spotting stuck sessions, checking whether something is currently running, and tracking what Alfred has been up to.

## How it works

ctrl-API shells into OpenClaw and runs `openclaw sessions`. JSON output is forwarded; non-JSON text is returned in a `raw` field.

<RequestExample>

```bash cURL
curl -X GET https://alfred.black/api/v1/openclaw/sessions \
  -H "Authorization: Bearer alf_your_key_here"
```

</RequestExample>

<ResponseExample>

```json Structured
{
  "sessions": [
    {
      "id": "sess-01HV...",
      "agent": "main",
      "status": "running",
      "started_at": "2026-04-25T08:14:02Z",
      "channel": "slack"
    },
    {
      "id": "sess-01HV...",
      "agent": "main",
      "status": "completed",
      "started_at": "2026-04-25T08:08:11Z",
      "channel": "email"
    }
  ]
}
```

```json Raw fallback
{
  "raw": "No active sessions"
}
```

</ResponseExample>

<Note>
  Ephemeral execution subagents spawned by the TaskRunner workflow live on the workers gateway (port 18790) and are not included in this list. The main openclaw gateway only tracks long-lived agents.
</Note>
