---
title: "Run doctor"
description: "Run OpenClaw's self-diagnostic and auto-repair routine."
api: "POST /api/v1/openclaw/doctor"
---

Runs the OpenClaw `doctor` command — a self-check that audits the gateway's configuration, verifies provider credentials, checks tool wiring, and attempts safe auto-fixes for common drift. Use it after a manual config edit, after upgrading the OpenClaw image, or when something feels off but `health` still reports green.

## How it works

ctrl-API shells into OpenClaw and runs `openclaw doctor`. JSON output is forwarded; non-JSON text is returned in a `raw` field.

<RequestExample>

```bash cURL
curl -X POST https://alfred.black/api/v1/openclaw/doctor \
  -H "Authorization: Bearer alf_your_key_here"
```

</RequestExample>

<ResponseExample>

```json Structured
{
  "checks": [
    { "name": "config.parses",      "status": "pass" },
    { "name": "providers.reachable", "status": "pass" },
    { "name": "tools.allowlist",     "status": "pass" }
  ],
  "fixed": [],
  "warnings": []
}
```

```json Raw fallback
{
  "raw": "All checks passed"
}
```

</ResponseExample>

<Note>
  Doctor is read-mostly. Auto-fixes are limited to safe, reversible changes such as repairing a malformed tool entry or re-syncing the gateway token. It will never delete user data or rotate credentials.
</Note>
