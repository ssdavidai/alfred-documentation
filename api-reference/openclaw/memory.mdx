---
title: "Search agent memory"
description: "Query OpenClaw's per-agent memory store, optionally filtered by a search term."
api: "GET /api/v1/openclaw/memory"
---

Queries the OpenClaw gateway's agent memory. With no parameters, returns recent memory entries. With a `query` parameter, filters entries to those matching the search term. Use this to inspect what Alfred has remembered between sessions.

## Query parameters

<ParamField query="query" type="string">
  Free-text search term. When omitted, the full recent memory is returned.
</ParamField>

## How it works

ctrl-API shells into OpenClaw and runs `openclaw memory [query]`. JSON output is forwarded; non-JSON text is returned in a `raw` field.

<RequestExample>

```bash With a query
curl -X GET "https://alfred.black/api/v1/openclaw/memory?query=vault%20cleanup" \
  -H "Authorization: Bearer alf_your_key_here"
```

```bash Without a query
curl -X GET https://alfred.black/api/v1/openclaw/memory \
  -H "Authorization: Bearer alf_your_key_here"
```

</RequestExample>

<ResponseExample>

```json Structured
{
  "matches": [
    {
      "agent": "main",
      "snippet": "Sir prefers concise summaries with bullet points first",
      "scored_at": "2026-04-25T07:42:00Z"
    }
  ]
}
```

```json Raw fallback
{
  "raw": "No memory entries found"
}
```

</ResponseExample>

<Note>
  This is OpenClaw's own per-agent memory, not the vault's MEMORY.md or the learning pipeline's instincts. For vault-side facts and user instructions, browse the `instinct/` and `event/` records via the vault endpoints.
</Note>
