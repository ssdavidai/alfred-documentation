---
title: "List plugins"
description: "Show plugins loaded by the OpenClaw gateway."
api: "GET /api/v1/openclaw/plugins"
---

Lists OpenClaw plugins. Plugins are extensions the gateway loads at startup — additional model providers, custom tool implementations, or auxiliary integrations beyond the built-in set.

## How it works

ctrl-API shells into OpenClaw and runs `openclaw plugins`. JSON output is forwarded; non-JSON text is returned in a `raw` field.

<RequestExample>

```bash cURL
curl -X GET https://alfred.black/api/v1/openclaw/plugins \
  -H "Authorization: Bearer alf_your_key_here"
```

</RequestExample>

<ResponseExample>

```json Structured
{
  "plugins": []
}
```

```json Raw fallback
{
  "raw": "No plugins installed"
}
```

</ResponseExample>

<Note>
  Most Alfred deployments rely on the bundled OpenClaw build with no extra plugins. If this list is empty on your instance, that's expected.
</Note>
