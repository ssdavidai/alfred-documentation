---
title: "List cron jobs"
description: "Show OpenClaw's internal cron schedule."
api: "GET /api/v1/openclaw/cron"
---

Lists the cron-style jobs the OpenClaw gateway runs internally. These are gateway-owned schedules — distinct from Temporal schedules (which the alfred-learn worker runs) and from the user-facing chore schedules visible in the dashboard.

## How it works

ctrl-API shells into OpenClaw and runs `openclaw cron`. JSON output is forwarded; non-JSON text is returned in a `raw` field.

<RequestExample>

```bash cURL
curl -X GET https://alfred.black/api/v1/openclaw/cron \
  -H "Authorization: Bearer alf_your_key_here"
```

</RequestExample>

<ResponseExample>

```json Structured
{
  "jobs": []
}
```

```json Raw fallback
{
  "raw": "No cron jobs configured"
}
```

</ResponseExample>

<Note>
  Most Alfred deployments don't use OpenClaw cron — recurring agent work runs through Temporal schedules instead. To inspect those, use the workflows and schedules endpoints.
</Note>
