---
title: "List skills"
description: "Show every skill loaded into the OpenClaw workspace."
api: "GET /api/v1/openclaw/skills"
---

Lists the skills available to Alfred and the vault workers. Skills are the prompt-and-procedure bundles that tell each agent how to do its job — for example, `alfred-vault-operations` for the main agent, `vault-curator` for the curator worker, or any per-app Composio skill that's been generated when Sir connected an integration.

## How it works

ctrl-API shells into OpenClaw and runs `openclaw skills`. JSON output is forwarded; non-JSON text is returned in a `raw` field.

<RequestExample>

```bash cURL
curl -X GET https://alfred.black/api/v1/openclaw/skills \
  -H "Authorization: Bearer alf_your_key_here"
```

</RequestExample>

<ResponseExample>

```json Structured
{
  "skills": [
    { "name": "alfred-vault-operations", "agent": "main" },
    { "name": "alfred-chore-management", "agent": "main" },
    { "name": "alfred-email-channel", "agent": "main" },
    { "name": "vault-curator", "agent": "vault-curator" },
    { "name": "vault-janitor", "agent": "vault-janitor" },
    { "name": "vault-distiller", "agent": "vault-distiller" }
  ]
}
```

```json Raw fallback
{
  "raw": "vault-curator, vault-janitor, vault-distiller"
}
```

</ResponseExample>

<Note>
  Two sources populate this list. Vault-worker skills (`vault-curator`, `vault-janitor`, `vault-distiller`) come from the Alfred Python package. Main-agent skills (`alfred-*`) and per-app Composio skills come from the platform workspace template, copied in by the init container on every boot.
</Note>
