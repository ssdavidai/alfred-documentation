---
title: "Restart gateway"
description: "Restart the OpenClaw container."
api: "POST /api/v1/openclaw/restart"
---

Restarts the OpenClaw container in your Alfred instance. Use this after editing `openclaw.json` directly, after rotating provider credentials, or as a last-resort recovery if Alfred has stopped responding. All in-flight sessions are interrupted.

## How it works

ctrl-API runs `docker compose restart openclaw` against your tenant's compose file. The container comes back in roughly 30–60 seconds on a cx53 Hetzner VPS.

## Response

<ResponseField name="message" type="string">
  Confirmation that the restart command was issued. Restart itself happens asynchronously inside Docker.
</ResponseField>

<RequestExample>

```bash cURL
curl -X POST https://alfred.black/api/v1/openclaw/restart \
  -H "Authorization: Bearer alf_your_key_here"
```

</RequestExample>

<ResponseExample>

```json 200 OK
{
  "message": "OpenClaw container restarted"
}
```

</ResponseExample>

<Warning>
  Any active conversation Sir is having with Alfred will be cut. Background workflows that depend on the gateway will fail and retry until the gateway is back up. Avoid restarting during active onboarding or chore generation runs.
</Warning>

<Note>
  After issuing the restart, poll `GET /api/v1/openclaw/health` (or `/ready`) until the gateway returns success.
</Note>
