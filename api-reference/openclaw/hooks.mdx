---
title: "List hooks"
description: "Show event hooks registered with the OpenClaw gateway."
api: "GET /api/v1/openclaw/hooks"
---

Lists OpenClaw event hooks. Hooks fire on gateway events (a session starts, a tool call completes, an agent emits a particular signal) and run small handler scripts in response. Alfred uses one such hook — `alfred-learn-observer` — to capture user instructions, corrections, and confirmations into the learning pipeline.

## How it works

ctrl-API shells into OpenClaw and runs `openclaw hooks`. JSON output is forwarded; non-JSON text is returned in a `raw` field.

<RequestExample>

```bash cURL
curl -X GET https://alfred.black/api/v1/openclaw/hooks \
  -H "Authorization: Bearer alf_your_key_here"
```

</RequestExample>

<ResponseExample>

```json Structured
{
  "hooks": [
    { "name": "alfred-learn-observer", "events": ["message.user", "message.assistant"] }
  ]
}
```

```json Raw fallback
{
  "raw": "No hooks registered"
}
```

</ResponseExample>

<Note>
  Hook handlers must be written as ESM modules (`export default handler`). CJS handlers fail silently.
</Note>
