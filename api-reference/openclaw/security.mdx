---
title: "Security audit"
description: "Run OpenClaw's security review of its own configuration."
api: "GET /api/v1/openclaw/security"
---

Runs the OpenClaw `security` audit — checks the current allowlist, MCP server registrations, secret handling, and gateway exposure for anything that looks misconfigured or risky. A useful periodic check after Sir adds a new integration or rotates credentials.

## How it works

ctrl-API shells into OpenClaw and runs `openclaw security`. JSON output is forwarded; non-JSON text is returned in a `raw` field.

<RequestExample>

```bash cURL
curl -X GET https://alfred.black/api/v1/openclaw/security \
  -H "Authorization: Bearer alf_your_key_here"
```

</RequestExample>

<ResponseExample>

```json Structured
{
  "findings": [],
  "allowlisted_tools": 12,
  "mcp_servers": ["alfred-ctrl"],
  "providers_with_credentials": ["openrouter", "anthropic", "google", "xai"]
}
```

```json Raw fallback
{
  "raw": "Security audit complete: no issues found"
}
```

</ResponseExample>

<Note>
  This is a read-only audit. It will never modify configuration or revoke access on your behalf. To act on a finding, use the relevant credentials, devices, or admin endpoint.
</Note>
