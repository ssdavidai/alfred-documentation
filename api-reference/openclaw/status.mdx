---
title: "Gateway status"
description: "Detailed gateway state — version, configured providers, allowlisted tools."
api: "GET /api/v1/openclaw/status"
---

Returns a richer view of the OpenClaw gateway than `health`. Use this when you want to confirm the gateway loaded the expected configuration: which providers it sees credentials for, the count of allowlisted tools, the active default model, and similar diagnostics.

## How it works

ctrl-API shells into the OpenClaw container and runs `openclaw gateway status`. JSON output is parsed and forwarded. If the CLI emits text, it's returned in a `raw` field.

<RequestExample>

```bash cURL
curl -X GET https://alfred.black/api/v1/openclaw/status \
  -H "Authorization: Bearer alf_your_key_here"
```

</RequestExample>

<ResponseExample>

```json Structured
{
  "version": "2026.4.15",
  "default_model": "xai/grok-4",
  "providers": ["openrouter", "anthropic", "google", "xai"],
  "tools_allowlisted": 12,
  "mcp_servers": ["alfred-ctrl"]
}
```

```json Raw fallback
{
  "raw": "Gateway status: running"
}
```

</ResponseExample>

<Note>
  For dashboard-style probes during a known restart window, use `GET /api/v1/openclaw/ready`. For the canonical inventory of built-in and MCP tools, use `GET /api/v1/openclaw/allowed-tools`.
</Note>
