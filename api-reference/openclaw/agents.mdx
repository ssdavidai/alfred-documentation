---
title: "List agents"
description: "Show every agent registered in the OpenClaw config."
api: "GET /api/v1/openclaw/agents"
---

Lists the agents OpenClaw knows about — `main` (Alfred), the vault worker agents (`vault-curator`, `vault-janitor`, `vault-distiller`), the `learn-clerk` worker, and any ephemeral execution subagents that happen to be alive at the time of the call.

## How it works

ctrl-API shells into OpenClaw and runs `openclaw agents`. JSON output is forwarded; non-JSON text is returned in a `raw` field.

<RequestExample>

```bash cURL
curl -X GET https://alfred.black/api/v1/openclaw/agents \
  -H "Authorization: Bearer alf_your_key_here"
```

</RequestExample>

<ResponseExample>

```json Structured
{
  "agents": [
    { "id": "main", "model": "xai/grok-4", "ready": true },
    { "id": "learn-clerk", "model": "xai/grok-4-1-fast-reasoning", "ready": true },
    { "id": "vault-curator", "model": "xai/grok-4-1-fast-reasoning", "ready": true },
    { "id": "vault-janitor", "model": "xai/grok-4-1-fast-reasoning", "ready": true },
    { "id": "vault-distiller", "model": "xai/grok-4-1-fast-reasoning", "ready": true }
  ]
}
```

```json Raw fallback
{
  "raw": "No agents configured"
}
```

</ResponseExample>

<Note>
  For the higher-level Alfred-facing view (label, description, surveyor config, provider auth status), use [`GET /api/v1/admin/agents`](/api-reference/agents/list). That endpoint reads from `config.yaml` plus per-agent model status and is what the dashboard renders.
</Note>
