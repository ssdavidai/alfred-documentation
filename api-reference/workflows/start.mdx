---
title: "Start Workflow"
description: "Start a new workflow execution"
api: "POST /api/v1/workflows"
---

Initiates a new workflow execution via `temporal workflow start`. Returns 201 with the Temporal CLI's parsed JSON output, or `{ "raw": "..." }` if the output is not valid JSON.

<ParamField body="workflow_type" type="string" required>
  The workflow type to execute. Must match a registered workflow name.
</ParamField>

<ParamField body="task_queue" type="string" required>
  The task queue where this workflow should be executed.
</ParamField>

<ParamField body="workflow_id" type="string" optional>
  Custom workflow ID. If omitted, a unique ID will be generated.
</ParamField>

<ParamField body="input" type="any" optional>
  Input data for the workflow. Will be JSON-serialized and passed to the Temporal CLI.
</ParamField>

<RequestExample>
```bash cURL
curl -X POST \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "workflow_type": "MyWorkflow",
    "task_queue": "my-task-queue",
    "workflow_id": "manual-run-001",
    "input": {
      "key": "value"
    }
  }' \
  https://app.alfred.black/api/v1/workflows
```
</RequestExample>

<ResponseExample>
```json 201
{
  "workflowId": "manual-run-001",
  "runId": "9b8e7f6a-5d4c-3b2a-1f0e-9d8c7b6a5f4e"
}
```
</ResponseExample>

<Note>
  The response structure is determined by the Temporal CLI's output. If the CLI outputs non-JSON text, the response will be `{ "raw": "..." }` instead.
</Note>
