---
title: "Signal Workflow"
description: "Send a signal to a running workflow"
api: "POST /api/v1/workflows/{wfId}/signal"
---

Send a signal to a running workflow. Signals allow you to communicate with a workflow while it is executing, triggering actions or passing updated data to it.

<ParamField path="wfId" type="string" required>
  The workflow ID to signal
</ParamField>

<ParamField body="signal_name" type="string" required>
  The name of the signal to send. Must match a signal handler defined in the workflow.
</ParamField>

<ParamField body="input" type="any" optional>
  Data to pass with the signal. Will be JSON-serialized.
</ParamField>

<ParamField body="run_id" type="string" optional>
  Specific run ID to signal. If omitted, signals the most recent run.
</ParamField>

<RequestExample>
```bash cURL
curl -X POST \
  -H "Authorization: Bearer alf_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{
    "signal_name": "update_config",
    "input": {
      "sweep_interval": 600,
      "batch_size": 10
    }
  }' \
  /api/v1/workflows/curator-sweep-20260226-143022/signal
```
</RequestExample>

<ResponseExample>
```json 200
{
  "message": "Signal sent"
}
```
</ResponseExample>
