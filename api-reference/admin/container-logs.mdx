---
title: "Container logs"
description: "One-shot fetch of recent log lines from a single container."
api: "GET /api/v1/admin/containers/{service}/logs"
---

Returns a snapshot of the most recent log lines from one container. Backs onto `docker compose logs --tail <n> <service>`. This is the synchronous counterpart to [`GET /api/v1/logs`](/api-reference/logs/stream) — use it from scripts that don't speak Server-Sent Events, or when you just want a quick peek.

## Path parameters

<ParamField path="service" type="string" required>
  Compose service id. Must match `[a-zA-Z0-9_-]+`.
</ParamField>

## Query parameters

<ParamField query="tail" type="string" default="100">
  Number of lines to return from the tail. Must be a non-negative integer.
</ParamField>

## Response

<ResponseField name="logs" type="string">
  Concatenated log output, lines separated by `\n`. Includes the leading `<service> | ` prefix that `docker compose logs` adds.
</ResponseField>

## Errors

`400 Bad Request` — `service` contains invalid characters or `tail` isn't numeric.

<RequestExample>

```bash Default tail (100)
curl -X GET https://alfred.black/api/v1/admin/containers/alfred/logs \
  -H "Authorization: Bearer alf_your_key_here"
```

```bash Custom tail
curl -X GET "https://alfred.black/api/v1/admin/containers/openclaw/logs?tail=500" \
  -H "Authorization: Bearer alf_your_key_here"
```

</RequestExample>

<ResponseExample>

```json 200 OK
{
  "logs": "alfred  | 2026-04-25T08:14:02Z INFO curator watching inbox/\nalfred  | 2026-04-25T08:14:03Z INFO janitor scan complete\n"
}
```

</ResponseExample>

<Note>
  For long-running tailing or live monitoring, use [`GET /api/v1/logs?service=...`](/api-reference/logs/stream). That SSE stream is what the dashboard's live log viewer uses.
</Note>
