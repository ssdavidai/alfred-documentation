---
title: "List containers"
description: "Show every container in the tenant Docker stack with state and health."
api: "GET /api/v1/admin/containers"
---

Returns the status of every container in your Alfred stack. Backs onto `docker compose ps --format json` against the tenant compose file. Useful for the dashboard's instance overview, scripted health monitoring, and post-deploy verification.

## Response

An array of compose container records. Field names are PascalCase straight from Docker Compose.

<ResponseField name="Name"     type="string">Full container name (e.g. `compose-openclaw-1`).</ResponseField>
<ResponseField name="Service"  type="string">Compose service id (`alfred`, `openclaw`, `alfred-learn`, `temporal`, `init`, `ctrl-api`, `openclaw-workers`, `ollama`).</ResponseField>
<ResponseField name="State"    type="string">`running`, `exited`, `restarting`, etc.</ResponseField>
<ResponseField name="Health"   type="string">Container health (`healthy`, `unhealthy`, `starting`) when a healthcheck is defined; empty string otherwise.</ResponseField>
<ResponseField name="ExitCode" type="integer">Exit code if the container has exited; `0` while running.</ResponseField>

<RequestExample>

```bash cURL
curl -X GET https://alfred.black/api/v1/admin/containers \
  -H "Authorization: Bearer alf_your_key_here"
```

</RequestExample>

<ResponseExample>

[

  { "Name": "compose-openclaw-1",        "Service": "openclaw",        "State": "running", "Health": "healthy",  "ExitCode": 0 },
  { "Name": "compose-openclaw-workers-1","Service": "openclaw-workers","State": "running", "Health": "",         "ExitCode": 0 },
  { "Name": "compose-alfred-1",          "Service": "alfred",          "State": "running", "Health": "",         "ExitCode": 0 },

<Note>
  The `init` service is a one-shot scaffolder and is expected to show `State: "exited"` with `ExitCode: 0` after a successful run. Anything else there indicates an init failure to investigate.
</Note>
